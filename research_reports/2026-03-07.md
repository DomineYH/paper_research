# 심층 연구 보고서 (2026-03-07)

오늘 수집된 논문군을 기반으로 다음 확장 주제를 통합 분석했다: (1) Agentic Context Engineering, (2) 저장소/에이전트 메모리 인프라, (3) 멀티모달 RAG 평가, (4) 도구 신뢰성·안전성 평가, (5) 한국어·국내 교육/도메인 적용.

## 박사학위급 연구주제 제안 A (핵심)
### **"지속형 컨텍스트 인프라와 적응형 RAG 게이팅을 결합한 멀티에이전트 코딩 시스템의 신뢰성 최적화"**

### 1) 이론적 배경
최근 연구는 에이전트 성능이 모델 파라미터보다 **컨텍스트 운영체계**에 크게 좌우됨을 시사한다. ACE는 컨텍스트를 자기개선 대상(진화 객체)으로 보고(Agentic Context Engineering: Evolving Contexts for Self-Improving Language Models, 2025), OSS 맥락 연구는 저장소 수준의 규범/제약 문서를 장기 기억 자산으로 본다(Context Engineering for AI Agents in Open-Source Software, 2025). 또한 AGENTS.md 효과 분석은 명시적 저장소 지침이 에이전트 효율을 개선할 수 있음을 보여준다(On the Impact of AGENTS.md Files on the Efficiency of AI Coding Agents, 2026).

이 흐름은 RAG 계열 평가 연구와 결합될 때 확장성이 커진다. RAG-Check는 검색 단계와 생성 단계를 분리해 오류책임을 추적하게 하고(RAG-Check, 2025), MMA-RAG는 내부 신뢰도에 따라 검색 활성화를 동적으로 조정하는 게이팅 전략을 제시한다(Multimodal Adaptive RAG, 2026). 따라서 “지속형 컨텍스트 인프라 + 적응형 검색 게이팅 + 분리평가”는 이론적으로 상호보완적이다.

### 2) 방법론 설계
- **시스템 아키텍처**
  - Layer 1: Repository Context Memory (AGENTS.md/규약/실패사례 로그)
  - Layer 2: Task-time Dynamic Context (현재 이슈, 테스트 결과, 변경 diff)
  - Layer 3: Adaptive RAG Gate (내부 불확실도 기반 외부 검색 on/off)
  - Layer 4: Verifier Agent (정적분석·테스트·정책검증)
- **실험군**
  1. Baseline 단일 에이전트(정적 프롬프트)
  2. 지속형 컨텍스트만 적용
  3. 컨텍스트 + 적응형 RAG 게이팅
  4. 컨텍스트 + 게이팅 + 검증 에이전트(완전체)
- **평가 데이터셋**
  - 오픈소스 이슈 해결 벤치(버그수정, 리팩터링, 테스트복구)
  - 멀티모달 문서(텍스트+표/다이어그램) 포함 과제
  - 한국어 혼합 과제(국내 적용성 확인; dCollection 한국어 수학 프롬프트 연구 연계)

### 3) 평가 프레임(정량/정성)
- **정량 지표**
  - 해결률, 평균 수정 라운드 수, 테스트 통과율, 회귀율
  - RAG-Check식 분리평가: Retrieval Relevancy / Generation Correctness
  - 장기작업 안정성: 세션 간 재실패율, 규약 위반율
- **정성 지표**
  - 개발자 신뢰도(설명가능성, 재현가능성)
  - 컨텍스트 자산 유지보수 비용
- **통계**
  - 과제 난이도 층화 후 mixed-effects model
  - 에이전트 설정 간 효과크기(Cohen’s d)와 비용-성능 프런티어 분석

### 4) 윤리·안전·거버넌스
- 컨텍스트 자산(AGENTS.md, 실패로그)에 민감정보/비밀키 유입 가능성 → 자동 비식별화/정책 스캐너 필요.
- 적응형 RAG는 외부 문서 오염 및 프롬프트 인젝션 위험을 내포 → 출처 신뢰도 가중치와 샌드박스 검증 필요.
- 교육 적용 시(국내 프롬프트 교육 연구) 과신·대리수행 부작용을 줄이기 위한 인간검토 단계 의무화 필요.

### 5) 산업적 파급
- 기업 코딩 에이전트 도입에서 핵심 KPI(속도·품질·재현성)를 동시에 개선할 가능성이 높다.
- “모델 교체 중심 전략”에서 “컨텍스트 운영 중심 전략”으로 투자 축이 이동할 수 있다.
- 한국어 업무 환경(문서/요구사항/규정이 한국어 중심)에서 국문 컨텍스트 자산의 표준화가 경쟁력이 된다.
