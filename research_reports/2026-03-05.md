# 심층 연구 보고서 (2026-03-05)

## 0) 연구 질문(박사학위급 도출)
**RQ:** *장기지평 멀티모달 에이전트에서 ‘메모리 제어(what to retain)–검색 정렬(what to retrieve)–안전 강건화(what to resist)’를 공동최적화할 때, 신뢰성(reliability) 지표를 실제 운영환경에서 유의미하게 향상시킬 수 있는가?*

이 질문은 당일 수집 논문의 공통 공백을 통합한다: (a) 장기 메모리 drift, (b) reason-to-retrieve 불일치, (c) 교차모달 공격 취약성, (d) 단일 성공률 중심 평가의 한계.

---

## 1) 이론적 분석
1. **기억의 표현 문제**: Memex(RL), CCS 계열 연구는 메모리를 단순 저장소가 아닌 정책 변수로 본다. 이는 전통 RAG의 정적 인덱스 가정과 충돌하며, 메모리 갱신 자체가 정책학습 대상이 된다.
2. **검색의 의도 정렬 문제**: AgentIR는 쿼리 텍스트보다 추론 문맥이 의도 신호에 가깝다고 본다. 즉, 검색 실패는 인덱스 품질보다 ‘의도 부호화 실패’일 수 있다.
3. **안전의 채널 확장 문제**: 멀티모달 웹 에이전트는 텍스트 방어만으로 충분치 않다. DOM-시각 동시 오염은 동일 거짓 내러티브를 이중 채널에 주입한다.
4. **평가의 다차원화 요구**: Reliability 연구는 성공률 평균이 운영 리스크를 숨긴다고 지적한다. 반복 일관성·실패 심각도·회복가능성 지표가 필요하다.

**종합 명제:** 차세대 에이전트 평가는 `Capability × Reliability × Safety`의 곱셈 구조로 설계돼야 하며, 셋 중 하나가 0에 가까우면 실사용 가치는 급격히 하락한다.

---

## 2) 방법론 설계(제안)
### 2.1 시스템 가설
- **H1**: Indexed memory policy(Memex류) + Reasoning-aware retrieval(AgentIR류) 결합은 장기 태스크 완주율을 유의하게 개선한다.
- **H2**: Cross-modal adversarial training(멀티모달 안전학습)을 병행하면 성능 손실 없이 실패 심각도를 완화한다.
- **H3**: ACE(컨텍스트 진화 루프)로 시스템 프롬프트·메모리 규칙을 주기적으로 재학습하면 drift 누적이 완화된다.

### 2.2 실험 프로토콜
- **Task set**: 심층 리서치, 웹 자동화, 다단계 도구사용, 한국어 프롬프트 공격 시나리오 포함
- **Ablation**: 
  - Base agent
  - + Memory policy
  - + Reasoning-aware retrieval
  - + Multimodal safety training
  - + ACE loop (full)
- **평가지표**:
  - Capability: task success, factuality
  - Reliability: rerun variance, perturbation robustness, recovery latency
  - Safety: jailbreak/prompt-injection success rate, harm severity
  - Cost: 토큰·지연·추론비용

### 2.3 국내 연구와의 연결
국내 KCI/DBpia/RISS 연구는 사용자 기대격차, 프롬프트 공격, 품질요인이 지속사용에 미치는 영향을 보여준다. 이는 국제 벤치마크가 놓치는 사회기술적 변수(신뢰·기대·수용)를 보완해, 한국어권 운영평가 셋 구축 필요성을 뒷받침한다.

---

## 3) 평가 관점의 비판적 논의
1. **벤치마크 과적합 위험**: RAG 성능 향상을 retrieval 개선으로 오인할 가능성(표현 품질 교란).
2. **안전-성능 트레이드오프 단순화 위험**: AAJR류 연구가 제안하듯, 방향성 정규화로 손실을 줄일 여지가 있음.
3. **운영지표 미비**: 사용자 관점(기대격차·실재감)이 결여되면 채택률 예측 실패 가능.

---

## 4) 윤리·거버넌스 분석
- **윤리**: 멀티모달 공격은 사용자 인지 한계를 이용하므로 설명가능한 경고 UX가 필수.
- **개인정보**: 장기 메모리 정책은 민감정보 보존 기간·삭제권 설계를 동반해야 함.
- **감사가능성**: Agentics 2.0류 관측가능성/타입안정성 프레임이 규제 대응에 유리.

---

## 5) 산업 적용 시사점
- **엔터프라이즈 에이전트 운영**: 메모리·검색·안전 모듈을 분리 설계하지 말고 동시 튜닝 파이프라인으로 운영.
- **MLOps/AgentOps**: 성공률 대시보드 외에 신뢰성 카드(reliability card) 표준화 필요.
- **국내 시장**: 한국어 프롬프트 공격 벤치와 사용자 기대격차 데이터를 결합한 지역 특화 평가체계가 경쟁력.

---

## 6) 결론
당일 증거를 종합하면, 에이전트 성능의 병목은 단일 모델 능력이 아니라 **메모리 제어–검색 정렬–안전 강건화의 분절 운영**에 있다. 박사학위급 후속 연구로는 이 세 축의 공동최적화 프레임을 구축하고, 한국어권 사용자 수용지표를 포함한 운영형 벤치마크를 제안하는 것이 타당하다.
