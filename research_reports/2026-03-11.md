# 심층 연구 보고서 (2026-03-11)

## 주제 제안(박사학위급)
**주제 A (핵심):**
**"동적 컨텍스트 엔지니어링과 JIT 메모리 컴파일을 결합한 신뢰가능 에이전트 아키텍처: 이론·평가·윤리·산업 전이 통합 연구"**

## 1. 연구 질문
1) 정적 메모리(사전 요약) 대비 동적/JIT 메모리 컴파일은 장기 과업에서 어떤 오류 유형을 줄이는가?  
2) 컨텍스트 파일(예: AGENTS.md) 품질과 에이전트 실행 신뢰성 사이의 인과관계는 어떻게 검증할 수 있는가?  
3) 멀티에이전트/멀티모달 환경에서 메모리 일관성과 도구 신뢰성을 동시 보장하는 평가 체계는 무엇인가?

## 2. 이론적 분석
- **형태-기능-동학 관점의 메모리 통합**: 최근 survey는 메모리를 토큰/파라메트릭/잠재 형태와 사실/경험/작업 기능으로 구분하고, 생성-진화-검색의 동학 루프를 명시한다(Hu et al., 2026).
- **정적 vs 동적 메모리 패러다임**: GAM은 오프라인 경량 메모리 + 온라인 JIT 재구성으로 정보손실을 완화하며, 질문 시점 적합도를 높인다(Liu et al., 2025).
- **컨텍스트 진화 설계**: ACE는 생성-반성-큐레이션 루프로 컨텍스트 붕괴를 방지하고 실행 피드백 기반 자기개선을 달성한다(Zhang et al., 2026).
- **국내 맥락 연결**: KCI/DBpia 연구들은 상태관리(State Manager), 프롬프트 구조, 사용자 상호작용 품질이 성능과 지속사용의도에 실질 영향을 준다는 정황을 제시한다(김/박, 2024~2025; RISS 학위논문).

## 3. 방법론 제안
### 3.1 아키텍처
- Layer 1: **Static Policy Context** (규범/제약/도메인 정책)
- Layer 2: **Adaptive Context Engine (ACE형)**
- Layer 3: **JIT Memory Compiler (GAM형 Memorizer+Researcher)**
- Layer 4: **Tool Reliability Guardrail** (실행 전후 검증, 재시도, provenance)

### 3.2 실험 설계
- 벤치마크: AppWorld류 에이전트 과업 + 시변 코퍼스 기반 RAG 평가(DRAGOn형)
- 조건군:
  - C0: 정적 프롬프트 + 단순 검색
  - C1: ACE만 적용
  - C2: JIT 메모리만 적용
  - C3: ACE + JIT + 도구 신뢰성 가드
- 지표:
  - 과업성공률, 장기 일관성, 오류복구율
  - 컨텍스트 붕괴율(반복 재작성 후 정보 보존율)
  - 근거 정합성(출처-응답 매칭률), 안전 위반율
  - 운영비용(토큰, 지연, 실패 재시도 비용)

## 4. 평가·검증 관점
- **RAG 평가 고도화**: 정적 데이터셋 점수 대신 주기 갱신 코퍼스에서 지속 성능을 측정해야 하며, 자동 질문 생성 + LLM-as-Judge를 병행하되 인간 검증 샘플링을 반드시 포함해야 한다(Fenogenova et al., 2026).
- **에이전트 메모리 평가 공백**: 메모리 품질은 저장량이 아니라 ‘적시성·관련성·오염 저항성’으로 측정되어야 한다(Hu et al., 2026; Du et al., 2025).
- **국내 교육/게임 실험 시사점**: 상태관리 루프 부재 시 조기 종료·주제 이탈·반복행동 증가가 관찰되어, 메모리 모듈을 정책 제어와 결합할 필요가 있다(KCI ART003228306; ART003214571).

## 5. 윤리·안전 분석
- **기억의 과잉 축적 위험**: 개인화 성능 향상과 프라이버시 침해 위험이 동반되므로, 메모리 최소수집·가중치 감쇠·삭제권 인터페이스가 필요(Memoria).
- **평가 자동화의 편향**: LLM-as-Judge 단독 평가는 모델 편향을 전이할 수 있어, 교차모델 평가 + 인적감사 샘플링을 포함해야 함(불확실성 존재).
- **산업 거버넌스**: 컨텍스트 파일 변경 이력(버전관리)과 실행 로그를 연결한 감사 추적이 필수(오픈소스 컨텍스트 엔지니어링 연구 시사점).

## 6. 산업 적용 시나리오
1) **엔터프라이즈 코딩 에이전트**: 대규모 코드베이스에서 장기 세션 안정성 및 도구 실패 회복 향상(Confucius Code Agent 맥락).  
2) **교육 진단 에이전트**: 학습자 상태 추적 + 회피 대화 복귀 전략으로 진단 누락 최소화(KCI 교육연구 맥락).  
3) **고객상담/비서형 AI**: 지식그래프 기반 장기 사용자 모델로 개인화 정밀도 향상(Memoria).

## 7. 결론
본일 수집 문헌은 공통적으로 **“정적 프롬프트 중심 설계에서, 동적 컨텍스트·메모리 운영 중심 설계로의 전환”**을 지시한다. 박사학위급 후속 연구는 단일 기술(프롬프트/메모리/RAG) 개선을 넘어, **컨텍스트-메모리-도구 신뢰성의 통합 제어 이론**과 **운영형(시간축) 평가 프로토콜**을 함께 제시해야 학술성과 산업재현성을 동시에 확보할 수 있다.
