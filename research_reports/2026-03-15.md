# 심층 연구 보고서 (2026-03-15)

## 연구 주제(박사학위급)
**"Context Engineering + Agent Memory + Agentic RAG의 통합 신뢰성 평가 프레임워크: 한국어 실무 데이터 환경에서의 다목적 최적화"**

---

## 1) 문제정의 및 이론적 배경
최근 에이전트 성능은 모델 자체보다 컨텍스트 설계, 메모리 운용, 검색 파이프라인 품질에 크게 의존한다는 근거가 축적되고 있다 (A Survey of Context Engineering for Large Language Models, 2025; Memory in the Age of AI Agents, 2025). 또한 agentic RAG는 정확도 향상 가능성을 보이지만 반복 루프에 따른 비용·지연·오류 전파 문제가 동반된다 (Agentic Retrieval-Augmented Generation, 2025). 국내 연구에서도 메타데이터 기반 검색 개선(M-RAG)과 self-corrective 다중 에이전트 접근이 신뢰성 향상에 기여할 가능성을 제시한다 (M-RAG, DBpia; Self-Corrective RAG, DBpia).

**핵심 가설 H1**: 한국어 지식업무 환경에서 성능 병목은 생성 모델의 파라미터 한계보다 "컨텍스트-메모리-검색" 결합 설계의 비최적성에서 더 크게 발생한다.

---

## 2) 방법론 제안
### 2.1 통합 아키텍처
- 계층 A: Context policy layer (정책 프롬프트 + 업무 규범 + 근거 요구 규칙)
- 계층 B: Hybrid memory layer (STM 대화기억 + LTM 사실기억 + 반성로그)
- 계층 C: Agentic retrieval layer (질의 재작성, 메타데이터 가중 검색, 증거 재검증)
- 계층 D: Reliability audit layer (툴호출 성공률, 출처정합성, 정책위반률)

이는 context engineering의 구조화 주장(2507.13334)과 메모리 계층화 주장(2512.13564), agentic RAG 루프 설계(2501.09136)를 결합한 설계다.

### 2.2 실험 설계
- 데이터: 한국어 기업 문서 + 교육 자료 + 규정 문서(비식별)
- 비교군:
  1) Baseline RAG
  2) Baseline + context templates
  3) Baseline + memory only (A-Mem/Mem0형)
  4) 제안모델(통합)
- 정량 지표:
  - 정확도(F1/EM)
  - 출처 정합률(citation grounding)
  - 정책위반률(safety/policy violation)
  - 툴호출 실패율
  - 비용(토큰/호출) 및 지연(latency)

---

## 3) 평가 관점(다각도)
### 3.1 이론
단일 지표 최적화가 아닌 다목적 최적화(정확도-신뢰성-비용)를 채택해야 한다. 이는 최근 에이전트 평가 서베이의 공통 결론과 일치한다 (AI Agent Systems, 2026; Agentic AI Taxonomy, 2026).

### 3.2 방법
국내 연구에서 확인된 메타데이터 기반 검색 강화(M-RAG)와 자기교정 루프(Self-Corrective RAG)를 통합 아키텍처의 retrieval/audit 계층에 반영할 수 있다.

### 3.3 윤리/안전
멀티에이전트와 장기기억이 결합될수록 개인정보 잔존, 잘못된 기억 고착, 권한 오남용 위험이 증가한다. 따라서 기억 삭제권, 출처 추적성, 접근권한 분리가 필수다 (Memory in the Age of AI Agents, 2025; COMMA, 2024).

### 3.4 산업
국내 조직은 모델 교체보다 운영 파이프라인 개선으로 빠르게 ROI를 확보할 가능성이 높다. 특히 규정준수 산업(금융/의료/공공)은 self-corrective audit 루프 도입 효과가 클 것으로 예상된다(불확실성: 대규모 실증 데이터는 추가 확보 필요).

## 4) 검증 가능한 연구질문
1. 한국어 업무문서에서 메모리+컨텍스트 통합이 hallucination을 유의미하게 줄이는가?
2. 메타데이터 강화 검색은 정확도 향상 대비 지연 증가를 어느 수준까지 허용 가능한가?
3. 정책 위반 탐지율과 사용자 만족도 간의 trade-off는 어떻게 최적화되는가?
4. 멀티모달 증거(표/이미지 포함) 환경에서 audit trace 품질은 어떻게 변하는가?
