## 1) A Systematic Survey of Prompt Engineering in Large Language Models: Techniques and Applications
- 링크: https://arxiv.org/abs/2402.07927
- DOI: 미확인(arXiv preprint)
- 요약(약 500자): 이 논문은 프롬프트 엔지니어링을 단순 입력 문구 최적화가 아니라, 지시문 설계·예시 구성·추론 유도·도구 결합을 포함한 체계적 방법론으로 재정의한다. 특히 LLM/VLM 전반에서 활용되는 기법을 분류하고, 과제 유형별로 어떤 프롬프트 패턴이 안정적으로 작동하는지 비교한다. 또한 평가 축을 정확도뿐 아니라 견고성·전이 가능성·비용 효율성으로 확장해, 실제 서비스 환경에서의 선택 기준을 제시한다. 연구적 기여는 “기법 백과사전”이 아니라 설계-평가-개선 루프를 제시했다는 점이다.
- 해시태그: #PromptEngineering #LLM #Survey #Evaluation #ContextDesign

## 2) The Prompt Report: A Systematic Survey of Prompt Engineering Techniques
- 링크: https://arxiv.org/abs/2406.06608
- DOI: 미확인(arXiv preprint)
- 요약(약 500자): Prompt Report는 프롬프트 엔지니어링 실무를 재현 가능하게 만들기 위해 기법, 실험 절차, 실패 양상을 하나의 프레임으로 통합한다. 논문은 제로샷·퓨샷·역할지정·CoT·구조화 출력 등 빈출 전략을 정리하고, 과제 난이도·모델 규모·출력 형식 제약에 따라 전략의 유효 범위가 달라짐을 강조한다. 특히 “좋은 프롬프트”를 결과물의 품질만으로 평가하지 않고, 일관성·오류 유형·운영 비용까지 포함해 판단해야 한다는 점을 제시한다. 팀 단위 운영에서는 프롬프트 버전관리와 로그 기반 회고가 핵심이라는 실무적 함의가 크다.
- 해시태그: #PromptReport #Reproducibility #ChainOfThought #PromptOps #LLMEvaluation

## 3) Reverse Prompt Engineering
- 링크: https://arxiv.org/abs/2411.06729
- DOI: 미확인(arXiv preprint)
- 요약(약 500자): Reverse Prompt Engineering은 모델 출력만 관찰 가능한 블랙박스 조건에서 원래 입력 프롬프트를 역추정하는 문제를 다룬다. 제안 방식은 추가 학습 없이 LLM과 최적화 절차를 결합해 후보 프롬프트를 생성·정련하고, 출력 유사도 기반으로 복원 품질을 높인다. 이 연구는 공격/유출 관점뿐 아니라, 에이전트 품질감사와 실패 포렌식에도 활용 가능하다는 점이 중요하다. 즉 “왜 이런 결과가 나왔는가”를 입력 설계 수준에서 추적할 수 있어, 운영 환경의 관측 가능성(Observability)을 높인다. 동시에 프롬프트 보안과 정책적 보호 필요성도 함께 제기한다.
- 해시태그: #ReversePrompt #ModelInversion #PromptSecurity #LLMAudit #Observability

## 4) Prompt Engineering and the Effectiveness of Large Language Models in Enhancing Human Productivity
- 링크: https://arxiv.org/abs/2507.18638
- DOI: 미확인(arXiv preprint)
- 요약(약 500자): 본 연구는 프롬프트 구조의 명확성, 맥락 정보의 밀도, 출력 형식 제약이 인간 생산성에 미치는 영향을 실무 시나리오 중심으로 분석한다. 핵심 메시지는 “모델 성능”과 “사용자 생산성”이 동일하지 않다는 점이다. 즉, 정답률이 비슷해도 프롬프트가 작업 흐름에 맞게 설계되면 수정 시간·재질문 횟수·오류 후처리 비용이 크게 감소한다. 이는 데이터 분석 보고, 문서 생성, 의사결정 보조 같은 반복 업무에서 더욱 두드러진다. 결과적으로 프롬프트 엔지니어링은 모델을 똑똑하게 만드는 기술이면서 동시에 인간-모델 협업 인터페이스를 최적화하는 생산성 공학으로 해석된다.
- 해시태그: #HumanProductivity #PromptDesign #HumanAIInteraction #WorkflowAI #LLM

## 5) Universal Conditional Logic: A Formal Language for Prompt Engineering
- 링크: https://arxiv.org/abs/2601.00880
- DOI: 미확인(arXiv preprint)
- 요약(약 500자): Universal Conditional Logic(UCL)은 프롬프트 설계를 직관과 시행착오에 의존하던 방식에서 벗어나, 형식 언어와 제약 기반 최적화로 다루려는 시도다. 논문은 조건-규칙-목표를 명시적으로 표현하는 언어적 틀을 제안해, 프롬프트 구성요소 간 상호작용을 분석 가능하게 만든다. 이는 복합 태스크에서 흔한 모호성·충돌·누락 문제를 줄이고, 실험 재현성을 높이는 데 유리하다. 특히 에이전트 파이프라인처럼 단계가 많은 시스템에서 규칙 기반 점검이 가능해져 품질 보증에 기여한다. 장기적으로는 프롬프트 컴파일러/검증기 같은 자동화 도구의 기반이 될 잠재력이 있다.
- 해시태그: #FormalMethods #PromptLanguage #Optimization #AIEngineering #Reliability

## 6) Agentic Context Engineering: Evolving Contexts for Self-Improving Language Models
- 링크: https://arxiv.org/abs/2510.04618
- DOI: 미확인(arXiv preprint)
- 요약(약 500자): 이 논문은 모델 가중치 업데이트 없이도 성능을 개선할 수 있는 핵심 메커니즘으로 “컨텍스트 진화”를 제시한다. 즉, 작업 지시·증거·전략·이전 실패 이력 등 컨텍스트를 동적으로 재구성해 에이전트가 스스로 개선되도록 설계한다. 이는 단발성 프롬프트 최적화보다 장기 과제에서 효과가 크며, 특히 도구 호출과 다단계 추론이 필요한 업무에서 실패 복구력 향상에 기여한다. 논문의 실무적 의의는 컨텍스트를 정적 텍스트가 아닌 운영 가능한 상태(state)로 본다는 점이다. 결과적으로 에이전트 시스템은 모델 튜닝보다 컨텍스트 설계 역량이 성능 상한을 결정할 가능성이 높다.
- 해시태그: #AgenticAI #ContextEngineering #SelfImprovingAgents #ToolUse #StateManagement

## 7) You Don’t Need Prompt Engineering Anymore: The Prompting Inversion
- 링크: https://arxiv.org/abs/2510.22251
- DOI: 미확인(arXiv preprint)
- 요약(약 500자): 본 논문은 기존 CoT 중심 프롬프트 엔지니어링의 한계를 지적하며, 규칙 기반 구조화 접근(예: Sculpting)을 통해 오류를 줄이는 대안을 제시한다. 핵심 주장은 “프롬프트를 더 길게 쓰는 것”보다 추론 경로를 명시적으로 제한·유도하는 설계가 복잡 문제에서 안정적일 수 있다는 것이다. 또한 의미적 모호성, 상식 추론의 비일관성, 장문 컨텍스트 왜곡 같은 실패 요인을 체계적으로 다룬다. 다만 제목의 강한 주장과 달리, 실무에서는 프롬프트 엔지니어링의 완전한 대체보다 방법론의 전환으로 해석하는 것이 타당하다. 즉 자유형 프롬프트에서 제약형 설계로의 이동을 시사한다.
- 해시태그: #PromptInversion #ReasoningControl #Sculpting #CoT #Robustness

## 8) ReAct: Synergizing Reasoning and Acting in Language Models
- 링크: https://arxiv.org/abs/2210.03629
- DOI: 10.48550/arXiv.2210.03629
- 요약(약 500자): ReAct는 “생각(Reasoning)”과 “행동(Acting)”을 교차시키는 프롬프트 패턴을 제안해, LLM이 외부 도구 사용과 중간 추론을 결합하도록 유도한다. 이 방식은 단순 응답 생성보다 검색·검증·의사결정 연쇄가 필요한 태스크에서 특히 강점을 보인다. 데이터 과학 맥락에서는 질의-분석-검증 단계를 명시적으로 기록할 수 있어 오류 추적이 용이하고, 재현 가능한 워크플로우 구축에 유리하다. 또한 실패 시 어느 단계에서 문제가 발생했는지 파악하기 쉬워 운영 안정성 개선에 기여한다. 오늘날 에이전트 프레임워크의 기본 설계 원리 중 하나로 자리 잡은 이유다.
- 해시태그: #ReAct #AIAgent #ToolAugmentedLLM #ReasoningActing #DataScienceWorkflow

## 9) Toolformer: Language Models Can Teach Themselves to Use Tools
- 링크: https://arxiv.org/abs/2302.04761
- DOI: 10.48550/arXiv.2302.04761
- 요약(약 500자): Toolformer는 LLM이 계산기·검색·번역기 등 외부 도구를 언제 호출해야 하는지 스스로 학습하도록 설계된 접근이다. 핵심은 인간이 일일이 규칙을 작성하지 않아도, 모델이 도구 호출이 유리한 문맥을 데이터 기반으로 발견하도록 만드는 데 있다. 이는 에이전트 운영에서 프롬프트 복잡도를 줄이고, 도구 사용의 일관성을 높이는 효과가 있다. 특히 데이터 분석 업무에서 수치 계산, 사실 확인, 지식 갱신을 분리해 처리함으로써 환각(hallucination) 위험을 완화한다. 결과적으로 고품질 출력은 모델 내부 추론만이 아니라 외부 도구 생태계와의 조합에서 나온다는 점을 보여준다.
- 해시태그: #Toolformer #ToolUse #AIAgents #HallucinationReduction #LLMSystems

## 10) Unleashing the Potential of Prompt Engineering for Large Language Models (rev. 2025)
- 링크: https://arxiv.org/abs/2310.14735
- DOI: 10.48550/arXiv.2310.14735
- 요약(약 500자): 이 논문은 프롬프트 엔지니어링을 구성요소 단위로 분해해, 어떤 요소가 어떤 과제에서 성능 향상에 기여하는지 폭넓게 정리한다. 최신 개정본은 다양한 응용 분야 사례를 보강하며, 단일 프롬프트 기법의 우열보다 문제 특성·모델 특성·평가 기준의 적합성이 더 중요함을 강조한다. 특히 실무적 관점에서 프롬프트를 “한 번의 정답”이 아니라 지속 개선되는 설계 산출물로 보고, 테스트-로그-회고 기반 운영을 제안한다. 이는 컨텍스트 엔지니어링과 결합될 때 더 큰 효과를 내며, 에이전트 시스템의 신뢰성과 재사용성을 높이는 방향으로 연결된다.
- 해시태그: #PromptEngineering #BestPractices #LLMApplications #PromptLifecycle #ContextAwareAI
