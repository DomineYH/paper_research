# arXiv 논문 정리 (2026-03-05)

오늘 제안/채택한 확장 주제: **AI safety/evals, agent memory, tool-use reliability**

## 1) Agentic Context Engineering: Evolving Contexts for Self-Improving Language Models
- 링크: https://arxiv.org/abs/2510.04618
- DOI: 미확인(arXiv preprint)
- 요약: 본 논문은 모델 파라미터를 고정한 채로 컨텍스트를 지속적으로 개선하는 방식이 에이전트 성능을 크게 끌어올릴 수 있음을 보인다. 핵심은 메모리, 힌트, 근거, 전략 프롬프트를 정적 텍스트가 아니라 동적 상태로 다루는 점이다. 이는 장기 태스크에서 오류 누적을 줄이고, 실패 이후 복구를 빠르게 하며, 도메인별 추론 성능을 안정화한다. 특히 prompt engineering을 단건 최적화가 아닌 운영 파이프라인으로 전환해야 한다는 실무적 함의를 제공한다.
- 해시태그: #ContextEngineering #AIAgent #AgentMemory #SelfImprovement

## 2) Reverse Prompt Engineering
- 링크: https://arxiv.org/abs/2411.06729
- DOI: 미확인(arXiv preprint)
- 요약: 이 연구는 모델 출력만 관찰 가능한 블랙박스 조건에서 원 프롬프트를 역추정하는 문제를 다룬다. 학습 없이 후보 프롬프트를 반복 생성·정련하는 방식으로 복원 품질을 높이며, 보안 측면뿐 아니라 감사/디버깅 용도로도 가치가 있음을 보여준다. 에이전트 운영에서는 오류 결과를 역으로 추적해 입력 설계 결함을 찾는 포렌식 도구로 해석할 수 있다. 프롬프트 유출 위험과 함께 관측가능성 향상이라는 양면성을 동시에 제시한다.
- 해시태그: #ReversePrompt #PromptSecurity #LLMAudit #Observability

## 3) The Prompt Report: A Systematic Survey of Prompt Engineering Techniques
- 링크: https://arxiv.org/abs/2406.06608
- DOI: 미확인(arXiv preprint)
- 요약: Prompt Report는 프롬프트 기법을 실험 단위로 구조화해 재현 가능한 실무 적용 기준을 제공한다. 제로샷, 퓨샷, CoT, 구조화 출력 제약 등의 장단점을 과제 유형별로 정리하며, 성능평가를 정확도뿐 아니라 일관성·비용·실패 유형으로 확장해야 함을 강조한다. 팀 단위 운영에서는 프롬프트 버전관리와 로그 기반 회고 체계가 필수라는 점을 분명히 한다. 오늘 파이프라인의 품질 규칙(중복 제거, DOI 체크, 인용 정합성)과 직접 연결되는 기준 문헌이다.
- 해시태그: #PromptEngineering #PromptOps #Reproducibility #LLMEvaluation

## 4) Prompt Engineering and the Effectiveness of LLMs in Enhancing Human Productivity
- 링크: https://arxiv.org/abs/2507.18638
- DOI: 미확인(arXiv preprint)
- 요약: 본 논문은 프롬프트 구조의 명료성, 과업 정의의 구체성, 출력 포맷 제한이 실제 업무 생산성에 미치는 영향을 분석한다. 중요한 결론은 모델 점수가 높아도 사용자 생산성이 반드시 높아지지 않는다는 점이다. 반대로 명확한 지시와 검증 가능한 출력 형식을 제공하면 재질문 횟수, 후편집 시간, 오류 수정 비용이 줄어든다. 데이터 과학 리포팅처럼 반복 작업이 많은 환경에서 prompt quality는 단순 성능이 아니라 운영 효율성 지표로 보아야 한다.
- 해시태그: #HumanProductivity #PromptDesign #WorkflowAI #DataScience

## 5) Universal Conditional Logic: A Formal Language for Prompt Engineering
- 링크: https://arxiv.org/abs/2601.00880
- DOI: 미확인(arXiv preprint)
- 요약: UCL은 프롬프트 엔지니어링을 형식화하려는 접근으로, 조건·규칙·목표를 명시적 논리 구조로 표현한다. 이 틀은 프롬프트 설계를 직관 중심에서 검증 가능한 공학 프로세스로 전환시키며, 다단계 에이전트에서 발생하는 모호성·충돌·누락을 줄이는 데 강점을 보인다. 장기적으로는 프롬프트 컴파일러, 정적검사, 자동 품질검증 도구로 확장될 가능성이 있다. 고신뢰 워크플로우를 원하는 조직에 특히 유의미한 방향이다.
- 해시태그: #FormalMethods #PromptLanguage #AIEngineering #Reliability

## 6) You Don’t Need Prompt Engineering Anymore: The Prompting Inversion
- 링크: https://arxiv.org/abs/2510.22251
- DOI: 미확인(arXiv preprint)
- 요약: 이 논문은 기존 CoT 중심 프롬프트 최적화가 가진 불안정성을 지적하고, 규칙 기반 추론 제약을 통해 오류를 줄이는 전략을 제시한다. 핵심은 프롬프트 길이를 늘리는 방식보다 추론 경로를 통제하는 설계가 복잡 과제에서 더 안정적일 수 있다는 주장이다. 다만 제목처럼 프롬프트 엔지니어링의 종말이라기보다, 자유형 설계에서 제약형 설계로의 이동으로 해석하는 것이 타당하다. 에이전트 안전성·검증성 요구가 높은 환경에서 실용성이 크다.
- 해시태그: #PromptInversion #ReasoningControl #AISafety #Robustness

## 7) ReAct: Synergizing Reasoning and Acting in Language Models
- 링크: https://arxiv.org/abs/2210.03629
- DOI: 10.48550/arXiv.2210.03629
- 요약: ReAct는 추론과 행동(도구 호출)을 교차시키는 프롬프트 구조를 제안해, 모델이 근거 탐색과 판단을 분리·연계하도록 만든다. 이 구조는 장문 생성보다 검색·검증·수정이 중요한 태스크에서 강력하며, 오류 발생 시 어느 단계에서 실패했는지 추적하기 쉽다. 데이터 과학 자동화에서 ReAct는 결과 재현성과 품질 감사의 기반 패턴으로 널리 쓰인다. 오늘 파이프라인의 tool-use reliability 확장 주제와 직접 연결되는 핵심 논문이다.
- 해시태그: #ReAct #ToolUse #AIAgent #Reliability

## 8) Toolformer: Language Models Can Teach Themselves to Use Tools
- 링크: https://arxiv.org/abs/2302.04761
- DOI: 10.48550/arXiv.2302.04761
- 요약: Toolformer는 모델이 외부 도구를 언제 호출할지 스스로 학습하도록 설계된 접근이다. 이 방식은 인간이 모든 도구 호출 규칙을 수작업으로 정의하지 않아도 되는 장점이 있으며, 계산·검색·사실검증을 분리해 환각을 줄인다. 에이전트 워크플로우에서 도구 선택이 곧 품질이라는 점을 실증적으로 뒷받침하며, tool reliability 연구의 기반이 된다. 실무에서는 프롬프트 설계와 도구 정책을 함께 설계해야 성능이 안정화된다는 교훈을 제공한다.
- 해시태그: #Toolformer #ToolReliability #HallucinationReduction #AgenticAI

## 9) A Systematic Survey of Prompt Engineering in LLMs
- 링크: https://arxiv.org/abs/2402.07927
- DOI: 미확인(arXiv preprint)
- 요약: 이 서베이는 프롬프트 엔지니어링의 핵심 구성요소를 체계적으로 분해하고 적용 영역별 설계 전략을 정리한다. 특히 instruction clarity, context packing, output schema의 결합이 성능 안정성에 미치는 영향을 강조하며, 단일 베스트 프롬프트보다 태스크 적합성과 평가 프로토콜이 중요하다고 본다. 연구·산업 현장에서 프롬프트를 문장 기술이 아닌 시스템 디자인으로 다뤄야 한다는 관점을 확립한다. 본 파이프라인의 기준 문헌으로 활용 가능하다.
- 해시태그: #PromptSurvey #LLM #ContextPacking #PromptStrategy

## 10) Unleashing the Potential of Prompt Engineering for LLMs
- 링크: https://arxiv.org/abs/2310.14735
- DOI: 10.48550/arXiv.2310.14735
- 요약: 본 논문은 다양한 프롬프트 전략을 통합적으로 비교하고, 과제 특성과 모델 특성에 따라 전략 효율이 달라짐을 보여준다. 특히 복합 태스크에서 프롬프트 구조·근거 제시·형식 제약의 결합이 결과 품질과 신뢰성을 동시에 높이는 경향을 보고한다. 또한 프롬프트 설계를 일회성 작업이 아니라 실험-평가-개선 사이클로 운영해야 한다는 점을 강조한다. 조직 관점에서는 prompt engineering을 운영 프로세스로 정착시키는 출발점으로 유용하다.
- 해시태그: #PromptLifecycle #LLMOps #Evaluation #ContextAware
