### 1) Agentic Context Engineering: Evolving Contexts for Self-Improving Language Models
- 링크: https://arxiv.org/abs/2510.04618
- DOI: 10.48550/arXiv.2510.04618
- 핵심기여: 컨텍스트(시스템 프롬프트/메모리/전략)를 **오프라인·온라인 동시 최적화** 대상으로 정의하고, 에이전트 성능을 반복적 진화 루프로 개선하는 ACE 프레임을 제시.
- 요약(약 500자): 본 논문은 LLM 성능 개선을 파라미터 업데이트 대신 컨텍스트 업데이트 문제로 재정의한다. 저자들은 정적 프롬프트 설계의 한계를 지적하며, 태스크 수행 로그와 실패 패턴을 반영해 컨텍스트를 주기적으로 재작성하는 ‘self-improving context loop’를 제안한다. 특히 에이전트 실행 중 축적되는 메모리와 규칙을 온라인으로 다듬고, 실험 세트 기반으로 오프라인 재학습을 병행해 성능 안정성과 일반화를 동시에 노린다. 요지는 “모델을 바꾸기 어렵다면, 모델이 읽는 맥락을 진화시켜라”이며, 실제 벤치마크에서 기준선 대비 향상을 보고한다.
- 키워드: #PromptEngineering #ContextEngineering #AIAgent #AgentMemory #SelfImprovement

### 2) Context Engineering for AI Agents in Open-Source Software
- 링크: https://arxiv.org/abs/2510.21413
- DOI: 10.48550/arXiv.2510.21413
- 핵심기여: OSS 개발환경에서 AI 코딩 에이전트 성능을 높이기 위한 **저장소-수준 컨텍스트 아키텍처**(규칙 문서, 빌드/테스트 지식, 운영 제약)를 체계화.
- 요약(약 500자): 논문은 프롬프트를 일회성 입력으로 다루는 접근이 대형 코드베이스에서 급격히 성능 저하를 유발한다고 본다. 대신 프로젝트 구조, 코딩 규약, 테스트 절차, 배포 제약을 지속적 문맥 자산으로 정리해 에이전트에게 주입하는 context engineering 방식을 제시한다. 핵심은 “개별 질의 최적화”보다 “저장소 전체의 규범·지식 전달”이며, 버전관리 가능한 텍스트 자산(예: 운영 가이드 파일)을 통해 재현성과 팀 협업성을 동시에 확보하는 점이다. 실무적으로는 장기 작업에서 오류 재발 감소와 작업 일관성 향상을 목표로 한다.
- 키워드: #AIAgent #ContextEngineering #SoftwareEngineering #ToolUse #RepositoryMemory

### 3) Codified Context: Infrastructure for AI Agents in a Complex Codebase
- 링크: https://arxiv.org/abs/2602.20478
- DOI: 10.5281/zenodo.18746623
- 핵심기여: 복잡한 코드베이스에서 에이전트 망각 문제를 줄이기 위해 **코드화된 컨텍스트 인프라**(지속적 규약·실패기록·작업플레이북) 필요성을 실증적으로 제시.
- 요약(약 500자): 저자들은 LLM 에이전트가 세션이 바뀌면 프로젝트 맥락을 잃고 동일 실수를 반복하는 문제를 다룬다. 이를 해결하기 위해 컨텍스트를 임시 대화가 아니라 인프라 계층으로 승격한다. 즉, 아키텍처 의사결정, 금지 패턴, 테스트 실패 교훈 등을 기계가 읽기 쉬운 형태로 축적하고, 작업 시작 시 자동으로 주입하도록 설계한다. 논문의 의의는 프롬프트 엔지니어링을 ‘문구 기술’에서 ‘지식 운영 체계’로 확장했다는 데 있다. 다만 DOI가 arXiv DOI가 아닌 Zenodo 메타데이터로 표기되어 있어 서지 추적 시 교차확인이 필요하다.
- 키워드: #AgentMemory #ContextInfrastructure #AICoding #Reliability #ContextEngineering

### 4) On the Impact of AGENTS.md Files on the Efficiency of AI Coding Agents
- 링크: https://arxiv.org/abs/2601.20404
- DOI: 10.48550/arXiv.2601.20404
- 핵심기여: AGENTS.md 같은 저장소 지침 파일이 코딩 에이전트 효율(반복수정, 오류율, 해결속도)에 미치는 영향을 분석.
- 요약(약 500자): 이 연구는 실무에서 급증한 AGENTS.md/CLAUDE.md류 파일의 효과를 정량적으로 조명한다. 핵심 가설은 명시적 지침 파일이 에이전트의 탐색 공간을 줄여, 불필요한 시행착오와 규약 위반을 감소시킨다는 것이다. 결과적으로 작업 완료까지의 라운드 수, 리그레션 위험, 컨텍스트 재설명 비용을 줄이는 방향의 근거를 제시한다. 특히 인간 개발자에게는 암묵지였던 규칙이 에이전트에게는 필수 입력이라는 점을 강조하며, 향후 에이전트 친화적 저장소 설계 원칙 수립의 기초 자료가 된다.
- 키워드: #AGENTSmd #AICodingAgent #ToolReliability #RepoOps #ContextEngineering

### 5) Context Engineering for Multi-Agent LLM Code Assistants Using Elicit, NotebookLM, ChatGPT, and Claude Code

- 요약(약 500자): 본 논문은 단일 LLM에 모든 책임을 맡기기보다, 문헌 탐색(지식 회수), 문서 통합(맥락 정리), 코드 생성, 검증을 분업한 multi-agent 설계를 제시한다. 이 접근은 컨텍스트 길이 한계와 도메인 지식 누락을 완화하며, 각 단계 산출물을 다음 단계의 구조화 입력으로 전달해 오류 전파를 줄인다. 특히 “좋은 답은 좋은 맥락 파이프라인에서 나온다”는 관점이 강하며, 코드 어시스턴트 성능을 모델 선택만이 아니라 입력 거버넌스 문제로 해석한다. 다만 실험 환경 이질성으로 일반화 범위는 추가 검증이 필요하다.
- 키워드: #MultiAgent #ContextPipeline #CodeAssistant #PromptEngineering #ToolChain

- 요약(약 500자): 이 논문은 프롬프트 품질과 인간 생산성 간 상관을 다룬다. 명확한 역할 지정, 작업 제약, 출력 포맷 요구가 포함된 프롬프트가 단순 질의 대비 더 높은 만족도와 효율을 보였다는 점을 보고한다. 연구의 실무 함의는 프롬프트 엔지니어링을 ‘부가 기술’이 아닌 협업 역량으로 보아야 한다는 데 있다. 특히 조직 도입 관점에서 템플릿 표준화, 태스크별 프롬프트 자산화, 반복 개선 사이클의 필요성을 뒷받침한다. 다만 설문 기반 분석 특성상 인과성 해석에는 주의가 요구된다.
- 키워드: #PromptEngineering #HumanProductivity #LLM #Workflow #Adoption

### 7) RAG-Check: Evaluating Multimodal Retrieval Augmented Generation Performance
- 링크: https://arxiv.org/abs/2501.03995
- DOI: 10.48550/arXiv.2501.03995
- 핵심기여: 멀티모달 RAG 평가에서 **검색 적합도(RS)**와 **생성 정합도(CS)**를 분리 측정하는 프레임 제안.
- 요약(약 500자): 논문은 RAG 실패 원인을 하나의 종합 점수로 묶으면 병목 위치를 놓치게 된다고 지적한다. 이에 검색 단계의 관련성(retrieval relevancy)과 답변 단계의 정확성(generation correctness)을 분리 평가하는 RAG-Check를 제안한다. 이 방식은 “검색은 맞았지만 생성이 틀린 경우”와 “검색 자체가 잘못된 경우”를 구분해 개선 우선순위를 명확히 한다. 멀티모달 환경(텍스트+비전)에서 특히 유용하며, 실무에서는 인덱서·리랭커·생성기 각 모듈의 책임 추적이 가능해진다.
- 키워드: #RAG #Multimodal #Evaluation #Reliability #Evals

### 8) Graph2Eval: Automatic Multimodal Task Generation for Agents via Knowledge Graphs
- 링크: https://arxiv.org/abs/2510.00507
- DOI: 10.48550/arXiv.2510.00507
- 핵심기여: 지식그래프 기반으로 멀티모달 에이전트 평가 태스크를 자동 생성해 데이터셋 확장성과 다양성 문제를 완화.
- 요약(약 500자): 이 연구는 정적 벤치마크가 빠르게 진화하는 에이전트 능력을 따라가지 못한다는 문제의식에서 출발한다. 저자들은 외부 데이터로 구성한 지식그래프에서 서브그래프를 샘플링해 태스크를 자동 생성하고, 도달가능성·난이도·유사도 필터를 거쳐 품질을 보장한다. 결과적으로 평가셋을 수작업 중심에서 반자동 생성 체계로 전환해 최신성 유지 비용을 낮춘다. RAG 에이전트와 웹 에이전트를 함께 다루는 점도 장점이며, 향후 에이전트 evals의 지속 업데이트 파이프라인 구축에 기여할 가능성이 크다.
- 키워드: #AgentEvals #KnowledgeGraph #MultimodalAgent #Benchmarking #AutoGeneration

### 9) MiRAGE: A Multiagent Framework for Generating Multimodal Multihop Question-Answer Dataset for RAG Evaluation
- 링크: https://arxiv.org/abs/2601.15487
- DOI: 10.48550/arXiv.2601.15487
- 핵심기여: 멀티에이전트 생성-검증 구조로 멀티홉·멀티모달 RAG 평가용 QA 데이터셋을 구축.
- 요약(약 500자): MiRAGE는 합성 데이터 품질 문제를 해결하기 위해 생성 에이전트와 검증 에이전트를 분리한 이중 구조를 채택한다. 현실 문서의 멀티모달 특성(텍스트, 표, 도표)을 반영하고, 단일 홉 질문이 아닌 복합 추론을 요구하는 멀티홉 질문을 생산한다는 점이 핵심이다. 이로써 기존 텍스트 중심 RAG 평가의 과소적합 문제를 보완하고, 실제 업무 환경에 가까운 난이도를 제공한다. 특히 검증기반 파이프라인은 환각 완화와 데이터셋 신뢰도 향상에 유리하지만, 검증 모델 자체의 편향 가능성은 후속 연구가 필요하다.
- 키워드: #RAGEvaluation #MultiAgent #MultihopQA #Multimodal #SyntheticData

### 10) Multimodal Adaptive Retrieval Augmented Generation through Internal Representation Learning
- 링크: https://arxiv.org/abs/2603.00511
- DOI: 10.48550/arXiv.2603.00511
- 핵심기여: 내부 표현 기반 신뢰도 추정으로 외부 검색 사용 여부를 동적으로 제어하는 적응형 멀티모달 RAG 제시.
- 요약(약 500자): 본 논문은 RAG가 항상 도움이 되지 않는다는 관찰에서 출발한다. 시각 질의응답 환경에서 유사하지만 오답인 외부 근거가 오히려 성능을 악화시키는 경우가 존재하며, 저자들은 모델 내부 표현을 이용해 ‘지금 검색을 붙일지’ 여부를 판단하는 적응형 전략을 제안한다. 즉, 내부 지식 신뢰가 높으면 생성만 수행하고, 불확실할 때만 검색을 활성화해 노이즈 유입을 줄인다. 이 접근은 정확도뿐 아니라 비용·지연 관리에도 장점이 있어 산업 적용성이 높다.
- 키워드: #AdaptiveRAG #Multimodal #InternalRepresentation #Reliability #ContextGating
