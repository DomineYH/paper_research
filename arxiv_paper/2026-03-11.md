### 1) Memory in the Age of AI Agents (arXiv:2512.13564)
- 링크: https://arxiv.org/abs/2512.13564
- DOI: 10.48550/arXiv.2512.13564
- 요약(약 500자): 본 논문은 급증하는 에이전트 메모리 연구를 ‘형태(forms)·기능(functions)·동학(dynamics)’의 3축으로 통합해 재정의한다. 기존의 장·단기 구분만으로는 최근 시스템 다양성을 설명하기 어렵다는 문제의식에서 출발해, 메모리 구현을 토큰/파라메트릭/잠재 형태로 분해하고, 기능을 사실·경험·작업 메모리로 세분화한다. 또한 형성-진화-검색의 시간적 루프를 설계 단위로 제시해, 단순 저장소 중심 설계를 넘어 메모리를 ‘에이전트 실행의 1급 추상’으로 위치시킨다. 벤치마크·오픈소스 프레임워크 지형도까지 함께 제시해 후속 연구의 기준 프레임을 제공한다.
- 핵심 기여: 메모리 개념 정합화, 3축 통합 분류체계, 실무형 벤치마크 맵 제시
- #키워드: #AgentMemory #ContextEngineering #AIAgent #Benchmark #Trustworthiness

### 2) Agentic Context Engineering: Evolving Contexts for Self-Improving Language Models (arXiv:2510.04618)
- 링크: https://arxiv.org/abs/2510.04618
- DOI: 10.48550/arXiv.2510.04618
- 요약: ACE는 컨텍스트를 고정 프롬프트가 아니라 ‘진화하는 플레이북’으로 다루는 프레임워크다. 생성-반성-큐레이션 모듈을 통해 실행 피드백을 흡수하고, 반복 요약 과정에서 발생하는 정보 소실(briefness bias)과 맥락 붕괴(context collapse)를 구조적으로 억제한다. 오프라인(시스템 프롬프트)과 온라인(에이전트 메모리) 양쪽에 적용 가능하며, 보고된 결과에서 에이전트 벤치마크와 도메인 태스크 모두 성능 개선 및 적응 비용 감소를 보였다. 라벨 데이터 없이 실행 피드백만으로 적응 가능하다는 점은 산업 적용 시 데이터 의존도를 낮추는 장점이 있다.
- 핵심 기여: 컨텍스트 진화 파이프라인, 붕괴 방지 구조, 비용-성능 동시 개선
- #키워드: #PromptEngineering #ContextEngineering #SelfImprovingAgents #AppWorld

### 3) A-MEM: Agentic Memory for LLM Agents (arXiv:2502.12110)
- 링크: https://arxiv.org/abs/2502.12110
- DOI: 10.48550/arXiv.2502.12110
- 요약: A-MEM은 Zettelkasten 원리를 에이전트 메모리 운영으로 확장해, 메모를 단순 로그가 아닌 연결 가능한 지식 노드로 조직한다. 새 기억이 들어오면 구조화된 노트를 만들고 과거 노드와 관계를 동적으로 재설정해, 기억망이 시간에 따라 재해석·재정렬되도록 설계한다. 기존 메모리 시스템의 고정 구조/고정 연산 한계를 넘어 태스크 맥락별 적응성을 강화하며, 다중 기반모델 실험에서 기존 대비 성능 향상을 보고한다. 본 접근은 회상 품질뿐 아니라 ‘메모리 갱신 규칙 자체’를 에이전트화했다는 점에서 의미가 크다.
- 핵심 기여: 동적 인덱싱·링킹, 메모리 진화(update) 메커니즘, 범용성 향상
- #키워드: #AgenticMemory #Zettelkasten #LLMAgent #LongHorizon

### 4) Context Engineering for AI Agents in Open-Source Software (arXiv:2510.21413)
- 링크: https://arxiv.org/abs/2510.21413
- DOI: 10.48550/arXiv.2510.21413
- 요약: 본 연구는 오픈소스 466개 프로젝트를 분석해 AGENTS.md류 컨텍스트 파일의 실제 채택 양상과 진화 패턴을 조사한다. 결과적으로 내용 구조 표준은 아직 약하며, 지시 문구가 기술/규범/금지/조건형 등 다양한 서술 방식으로 혼재함을 보인다. 이는 ‘좋은 컨텍스트’가 단일 템플릿 문제가 아니라 프로젝트 문화·리스크·작업유형에 따라 달라지는 설계 문제임을 시사한다. 특히 커밋 단위 변화 분석을 통해 컨텍스트 파일이 정적 문서가 아니라 운영 중 지속적으로 튜닝되는 아티팩트임을 보여준다.
- 핵심 기여: 실증적 AGENTS.md 채택 분석, 컨텍스트 파일 진화 관찰, 표준화 과제 도출
- #키워드: #AGENTSmd #SoftwareEngineering #ContextEngineering #Tooling

### 5) Confucius Code Agent: Scalable Agent Scaffolding for Real-World Codebases (arXiv:2512.10398)
- 링크: https://arxiv.org/abs/2512.10398
- DOI: 10.48550/arXiv.2512.10398
- 요약: CCA는 대형 저장소·장기 세션·복합 툴체인을 전제로 한 소프트웨어 에이전트 아키텍처를 제안한다. 핵심은 통합 오케스트레이터, 장문맥 관리, 세션 간 노트 지속성, 확장 모듈 기반 툴 사용 안정화다. 더불어 build-test-improve 메타에이전트를 통해 에이전트 자체를 반복 개선하는 자동화 루프를 제공한다. 보고 성능(예: SWE-Bench-Pro)은 연구용 시스템의 투명성과 실무 시스템의 확장성 사이 간극을 좁히려는 시도로 해석할 수 있다. 대규모 현업 적용에서 신뢰성·관측가능성·제어가능성을 함께 추구한 점이 핵심이다.
- 핵심 기여: 대규모 코드베이스용 스캐폴딩, 메타에이전트 자동개선 루프, 실무형 운영성 강화
- #키워드: #CodingAgent #ToolReliability #LongContext #SWEBench

### 6) Memoria: A Scalable Agentic Memory Framework for Personalized Conversational AI (arXiv:2512.12686)
- 링크: https://arxiv.org/abs/2512.12686
- DOI: 10.48550/arXiv.2512.12686 (등록 대기 표기)
- 요약: Memoria는 세션 요약 기반 단기 기억과 가중 지식그래프 기반 장기 사용자 모델을 결합한 하이브리드 메모리 구조를 제시한다. 단기 일관성과 장기 개인화를 분리-연동해 토큰 제약 하에서도 맥락 유지와 맞춤 응답 품질을 동시 추구한다. 해석 가능한 구조(그래프) 덕분에 기억의 출처와 변화 경로를 추적하기 쉬워, 개인화 시스템의 책임성 요구에도 대응 가능하다. 산업 관점에서는 CRM·상담·개인비서형 대화 서비스로의 이전 가능성이 높다.
- 핵심 기여: 요약+지식그래프 결합, 개인화 가능한 장기 메모리, 해석성 강화
- #키워드: #PersonalizedAI #KnowledgeGraph #ConversationalAgent #Memory

### 7) Rethinking Memory in LLM based Agents (arXiv:2505.00675)
- 링크: https://arxiv.org/abs/2505.00675
- DOI: 10.48550/arXiv.2505.00675
- 요약: 이 논문은 메모리를 표현(파라메트릭/컨텍스추얼)과 연산(통합·업데이트·인덱싱·망각·검색·응축)으로 분해해, 메모리 연구를 기능 단위로 비교할 수 있는 분석틀을 제안한다. 기존 ‘어떤 앱에 썼는가’ 중심 서술을 넘어, 메모리 시스템의 원자적 설계 연산을 정의했다는 점이 중요하다. 해당 틀은 장기기억, 장문맥, 파라미터 수정, 다중 소스 메모리 등 핵심 연구축을 연결해 벤치마크 설계와 모듈 교체 실험을 체계화하는 데 유용하다.
- 핵심 기여: 메모리 원자연산 관점 도입, 비교 가능한 분류틀, 연구지형 구조화
- #키워드: #MemoryOps #Forgetting #Retrieval #AgentDesign

### 8) Intrinsic Memory Agents (arXiv:2508.08997)
- 링크: https://arxiv.org/abs/2508.08997
- DOI: 10.48550/arXiv.2508.08997
- 요약: 본 연구는 멀티에이전트 LLM 시스템의 일관성 붕괴 문제(역할 이탈·절차 불안정·문맥 한계)를 해결하기 위해 에이전트별 내재 메모리 구조를 제안한다. 핵심은 역할 정렬(role-aligned) 메모리를 유지하면서 태스크 관련 정보만 압축·유지해 협업 충돌을 줄이는 것이다. 문제별 수작업 프롬프트 없이 범용 템플릿으로 이식 가능하다는 점에서 운영 비용도 낮춘다. 계획/검증 태스크에서 일관성 개선을 보였다는 보고는 멀티에이전트 시스템의 신뢰성 향상 관점에서 의미가 크다.
- 핵심 기여: 역할 정렬형 메모리, 범용 템플릿 기반 이식성, 멀티에이전트 일관성 개선
- #키워드: #MultiAgent #StructuredMemory #Consistency #Planning

### 9) General Agentic Memory Via Deep Research (arXiv:2511.18423)
- 링크: https://arxiv.org/abs/2511.18423
- DOI: 10.48550/arXiv.2511.18423
- 요약: GAM은 정적 사전메모리의 정보손실 한계를 지적하고, JIT(Just-in-Time) 맥락 컴파일 접근을 제안한다. 오프라인에는 경량 메모리와 원문 페이지 저장소를 유지하고, 온라인 요청 시 Researcher가 관련 정보를 조합해 최적 컨텍스트를 재구성한다. 이 구조는 ‘기억을 미리 완성’하는 대신 ‘질문 시점에 재합성’하는 패러다임으로, 최신성·정합성·확장성의 균형을 도모한다. RL 기반 종단 최적화 가능성을 함께 제시해 메모리 운영을 정책학습 대상으로 확장했다.
- 핵심 기여: JIT 컨텍스트 컴파일, Memorizer/Researcher 듀오 구조, 정적메모리 한계 완화
- #키워드: #JITMemory #DeepResearch #ScalableAgents #RL

### 10) DRAGOn: Designing RAG On Periodically Updated Corpus (arXiv:2507.05713)
- 링크: https://arxiv.org/abs/2507.05713
- DOI: 10.48550/arXiv.2507.05713
- 요약: DRAGOn은 주기적으로 갱신되는 코퍼스에서 RAG 시스템을 공정하게 비교하기 위한 벤치마크 설계를 제안한다. 고정 데이터셋 평가가 야기하는 데이터 누수/과최적화 문제를 줄이기 위해, 버전드 데이터셋 생성과 자동 QA 생성 파이프라인(KG 추출 포함), LLM-as-a-Judge 지표 묶음, 공개 리더보드를 결합했다. 이는 RAG 평가를 ‘정적 점수’에서 ‘지속 운영 품질’로 이동시키는 접근으로, 에이전트 시스템의 시간축 신뢰성 검증에 직접 연결된다.
- 핵심 기여: 시변 코퍼스 대응 RAG 벤치마크, 자동 생성 파이프라인, 운영형 평가체계
- #키워드: #RAGEvaluation #Benchmark #DataLeakage #Leaderboard
