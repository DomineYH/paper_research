### 1) Context Engineering: From Prompts to Corporate Multi-Agent Architecture
- 링크: https://arxiv.org/abs/2603.09619
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 본 논문은 프롬프트 엔지니어링을 단일 질의 최적화의 기술로 제한하고, 에이전트 시대에는 더 넓은 정보 환경을 설계하는 `Context Engineering(CE)`이 독립 학문으로 필요하다고 주장한다. 저자는 CE의 품질 기준으로 관련성(relevance), 충분성(sufficiency), 격리(isolation), 경제성(economy), 출처성(provenance)을 제시하며, 컨텍스트를 에이전트의 운영체계(OS)에 비유한다. 더 나아가 의도공학(Intent Engineering)과 명세공학(Specification Engineering)을 계층화하여 기업형 멀티에이전트 도입의 성숙도 모델을 제안한다.
- 핵심 기여: CE를 조직 운영 관점(의도·정책·표준)으로 확장한 개념 프레임.
- 키워드: #ContextEngineering #AgentArchitecture #IntentEngineering #SpecificationEngineering #Provenance

### 2) Agentic Context Engineering: Evolving Contexts for Self-Improving Language Models
- 링크: https://arxiv.org/abs/2510.04618
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 본 연구는 파라미터 재학습 없이도 에이전트 성능을 높일 수 있는 핵심 축으로 컨텍스트의 동적 진화를 제시한다. 고정 프롬프트 대신 실행 중 컨텍스트(지침, 증거, 전략)를 갱신·재구성해 성능을 향상시키는 접근을 다루며, 자가개선(self-improving) 루프에서 컨텍스트 관리가 사실상 정책 업데이트 역할을 수행할 수 있음을 보인다. 특히 도구 사용·도메인 추론 상황에서 컨텍스트 구조화 방식이 결과 품질과 안정성에 결정적 영향을 준다는 점을 강조한다. (초록 전문은 외부 검색 스니펫 기반 요약)
- 핵심 기여: “가중치 업데이트 없는 에이전트 개선”을 컨텍스트 진화 문제로 정식화.
- 키워드: #AgenticContext #SelfImprovingAgents #Prompting #ContextAdaptation #LLM

### 3) Context Engineering for AI Agents in Open-Source Software
- 링크: https://arxiv.org/abs/2510.21413
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 오픈소스 개발 환경에서 AI 에이전트가 실제로 소비하는 컨텍스트(문서, 설정 파일, 예시 코드, 규칙 문서)를 체계적으로 분석하고, 단순한 프롬프트 튜닝보다 저장소 맥락의 선별·조합이 품질에 더 큰 영향을 줄 수 있음을 보인다. 논문은 개발 워크플로우에서 “어떤 정보를 어떤 순서와 형식으로 제공할지”를 공학적 문제로 다루며, 리포지토리 컨텍스트 파일(예: 가이드/규약 문서)의 실무적 가치와 한계를 함께 제시한다. (초록 일부는 검색 인덱스/버전 정보 기반)
- 핵심 기여: OSS 코딩 에이전트 맥락에서 컨텍스트 엔지니어링의 실증적 관찰.
- 키워드: #OpenSource #AIAgents #ContextSelection #SoftwareEngineering #RepositoryContext

### 4) Agentic Memory: Learning Unified Long-Term and Short-Term Memory Management for LLM Agents
- 링크: https://arxiv.org/abs/2601.01885
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): AgeMem은 LTM/STM을 분리된 모듈이 아니라 단일 정책 안의 도구 기반 행동으로 통합한다. 에이전트가 저장·검색·요약·삭제를 스스로 선택하도록 설계하고, 희소 보상 문제를 완화하기 위해 단계적 강화학습(3-stage)과 step-wise GRPO를 도입한다. 5개 장기 과업 벤치마크에서 강한 베이스라인 대비 성능·메모리 품질·컨텍스트 효율이 개선되었으며, “메모리 관리 자체를 학습 가능한 의사결정”으로 다룬 점이 핵심이다.
- 핵심 기여: 메모리 연산을 정책 내부 행위로 통합한 end-to-end 학습 프레임.
- 키워드: #AgentMemory #LongTermMemory #ShortTermMemory #ReinforcementLearning #LLMAgents

### 5) Memory in the Age of AI Agents
- 링크: https://arxiv.org/abs/2512.13564
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 대규모 서베이로서 에이전트 메모리 연구를 형태(forms)-기능(functions)-동학(dynamics) 축으로 재분류한다. 토큰/파라메트릭/잠재 메모리, 사실/경험/작업 메모리의 구분을 제안하고, RAG·컨텍스트 엔지니어링과의 경계도 명시한다. 또한 벤치마크·오픈소스 프레임워크 지형을 정리하면서 자동화, RL 통합, 멀티모달 메모리, 멀티에이전트 메모리, 신뢰성 이슈를 차기 연구 프런티어로 제시한다.
- 핵심 기여: 파편화된 메모리 연구를 통합하는 최신 개념 지도와 용어 정리.
- 키워드: #Survey #AgentMemory #RAG #ContextEngineering #Trustworthiness

### 6) Diagnosing Retrieval vs. Utilization Bottlenecks in LLM Agent Memory
- 링크: https://arxiv.org/abs/2603.02473
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 3×3 진단 실험(쓰기 전략 3종 × 검색 방식 3종)으로 메모리 성능 병목의 주된 위치를 분석한다. 결과는 쓰기(write)보다 검색(retrieval) 방식의 영향이 훨씬 크며, 정교한 요약형 저장보다 단순 raw chunk 저장이 비용 대비 우수하거나 동등할 수 있음을 보인다. 실패 사례 분석에서도 활용(utilization) 이전 단계인 검색 실패가 주 원인으로 나타나, 현 시점에서는 write-time 복잡화보다 retrieval 개선에 투자하는 것이 실효적이라는 결론을 제시한다.
- 핵심 기여: 메모리 파이프라인에서 병목 위치를 분리해 보여준 실증 진단 프레임.
- 키워드: #MemoryRetrieval #AgentEvaluation #BM25 #HybridReranking #MemoryPipeline

### 7) MemoryArena: Benchmarking Agent Memory in Interdependent Multi-Session Agentic Tasks
- 링크: https://arxiv.org/abs/2602.16313
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 기존 벤치마크가 회상과 행동을 분리 평가하는 한계를 지적하고, 다중 세션에서 기억-행동이 상호의존하는 과제를 위한 MemoryArena를 제안한다. 웹 내비게이션, 제약계획, 점진 탐색, 연속 추론 등에서 과거 행동의 증류 메모리를 미래 과업 해결에 재사용해야 하며, 기존 long-context 벤치마크에서 높은 성능을 보인 시스템도 이 설정에서는 크게 성능 저하를 보인다. 즉 “기억을 갖는 것”과 “기억을 활용해 연쇄 의사결정하는 것”의 간극을 측정한다.
- 핵심 기여: 다중세션·상호의존 과업 중심의 agent memory 평가 프레임워크.
- 키워드: #Benchmark #MemoryArena #MultiSessionAgents #EvaluationGap #AgentTasks

### 8) Hindsight is 20/20: Building Agent Memory that Retains, Recalls, and Reflects
- 링크: https://arxiv.org/abs/2512.12818
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): Hindsight는 증거(evidence)와 추론(inference)이 뒤섞이는 기존 메모리 설계의 취약점을 완화하기 위해, 사실·경험·개체 요약·신념 업데이트를 분리하는 구조적 메모리 네트워크를 제시한다. 장기 상호작용에서 정보 정리와 설명가능성(왜 그렇게 판단했는지 추적)을 강화하는 데 목적이 있으며, 단순 저장소가 아닌 반성(reflection) 가능한 추론 기반 인프라로 메모리를 재정의한다. (초록 상세는 검색 스니펫 기반 보완 요약)
- 핵심 기여: “보존-회상-반성”을 분리한 설명가능 메모리 구조 제안.
- 키워드: #ReflectiveMemory #ExplainableAgents #BeliefUpdate #StructuredMemory #LongHorizon

### 9) MemEvolve: Meta-Evolution of Agent Memory Systems
- 링크: https://arxiv.org/abs/2512.18746
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): MemEvolve는 에이전트가 경험만 누적하는 수준을 넘어, 메모리 아키텍처 자체를 메타-진화시키는 프레임워크다. EvolveLab 코드베이스를 통해 encode/store/retrieve/manage의 모듈 공간을 표준화하고, 다양한 과업·모델에서 전이 가능한 메모리 설계를 탐색한다. 보고된 실험에서 여러 에이전트 프레임워크의 성능 향상과 교차 과업 일반화가 확인되며, 정적 메모리 설계의 한계를 넘어선 적응형 메모리 공학 가능성을 보여준다.
- 핵심 기여: “경험 진화 + 메모리 구조 진화”를 결합한 메타 학습 접근.
- 키워드: #MetaEvolution #AdaptiveMemory #EvolveLab #Generalization #LLMAgents

### 10) Multi-Agent Memory from a Computer Architecture Perspective: Visions and Challenges Ahead
- 링크: https://arxiv.org/abs/2603.10062
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 본 포지션 페이퍼는 멀티에이전트 메모리를 컴퓨터 아키텍처 관점으로 재해석한다. 공유/분산 메모리 패러다임을 구분하고 I/O–캐시–메모리 3계층을 제안하며, 에이전트 간 캐시 공유 프로토콜과 구조화된 접근제어의 부재를 핵심 공백으로 지적한다. 궁극적으로 멀티에이전트 환경의 가장 시급한 난제를 메모리 일관성(consistency)으로 정의하고, 신뢰 가능하고 확장 가능한 협업 에이전트 설계를 위한 시스템 수준 연구 어젠다를 제시한다.
- 핵심 기여: 멀티에이전트 메모리 문제를 OS/아키텍처 레벨의 일관성 문제로 정식화.
- 키워드: #MultiAgentMemory #MemoryHierarchy #Consistency #AccessControl #SystemsAI
