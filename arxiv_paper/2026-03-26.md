## 1. Generalizable Self-Evolving Memory for Automatic Prompt Optimization (MemAPO)

**저자:** Guanbao Liang et al.
**게재일:** 2026-03-23 | **분류:** cs.CL
**DOI:** [10.48550/arXiv.2603.21520](https://doi.org/10.48550/arXiv.2603.21520)
**링크:** <https://arxiv.org/abs/2603.21520>

### 요약
기존 프롬프트 최적화 방법들은 특정 작업에 고정된 프롬프트를 탐색하는 데 국한되어, 이기종 쿼리 간 일반화와 시간 경과에 따른 재사용 지식 축적이 불가했다. 본 논문은 MemAPO라는 메모리 주도 프롬프트 최적화 프레임워크를 제안한다. 성공적 추론 궤적을 재사용 가능한 전략 템플릿으로 증류하고, 오류 발생 패턴을 구조화된 에러 패턴으로 조직화하는 이중 메모리 메커니즘을 활용한다. 새 프롬프트가 주어지면 관련 전략과 실패 패턴을 검색하여 효과적 추론을 촉진하고 알려진 오류를 억제하도록 프롬프트를 구성한다. 반복적 자기 성찰과 메모리 편집을 통해 지속 개선하며, 다양한 벤치마크에서 기존 기준선을 일관되게 초과하면서 최적화 비용을 크게 절감했다.

### 핵심 기여
- 프롬프트 최적화를 범용적이고 자기 진화하는 경험 축적으로 재개념화
- 전략 템플릿과 에러 패턴의 이중 메모리 아키텍처 설계
- 작업 간 지식 전이를 통한 최적화 비용 절감 효과 입증

`#PromptOptimization #AgentMemory #SelfEvolvingSystems #LLM`

## 2. Evaluating 5W3H Structured Prompting for Intent Alignment in Human-AI Interaction

**저자:** Gang Peng
**게재일:** 2026-03-19 | **분류:** cs.AI, H.5.2, I.2.7
**DOI:** [10.48550/arXiv.2603.18976](https://doi.org/10.48550/arXiv.2603.18976)
**링크:** <https://arxiv.org/abs/2603.18976>

### 요약
자연어 프롬프트는 사용자 실제 필요와 AI 시스템에 전달되는 의도 간의 불일치(intent transmission loss)를 자주 겪는다. 본 연구는 PPS(Prompt Protocol Specification)라는 5W3H 기반 구조적 의도 표현 프레임워크를 평가한다. 비즈니스, 기술, 여행 3개 도메인 60개 작업에서 DeepSeek-V3, Qwen-Max, Kimi 세 가지 모델로 540개 출력을 생성하고 LLM 판사가 평가했다. goal_alignment라는 사용자 의도 중심 평가 차원을 도입하여, 자연어 렌더링된 PPS가 단순 프롬프트 및 원본 JSON보다 우수함을 확인했다. PPS 효과는 작업 의존적: 고모호성 비즈니스 분석에서 큰 향상, 저모호성 여행 계획에서는 반전 현상 관찰. 회고 설문(N=20)에서 후속 프롬프트 66.1% 감소(3.33→1.13회) 보고.

### 핵심 기여
- goal_alignment 평가 차원 도입으로 사용자 의도 정렬 측정 혁신
- 구조적 의도 표현의 효과가 작업 모호성에 따라 달라짐을 규명
- LLM 평가에서 측정 비대칭성 존재 식별

`#PromptEngineering #IntentAlignment #HumanAI #StructuredPrompting #Evaluation`

## 3. AgentDS Technical Report: Benchmarking the Future of Human-AI Collaboration in Domain-Specific Data Science

**저자:** An Luo, Jin Du, Xun Xian et al.
**게재일:** 2026-03-19 | **분류:** cs.LG, cs.AI, stat.ME
**DOI:** [10.48550/arXiv.2603.19005](https://doi.org/10.48550/arXiv.2603.19005)
**링크:** <https://arxiv.org/abs/2603.19005>

### 요약
LLM과 AI 에이전트의 발전으로 데이터 과학 워크플로우가 크게 자동화되었으나, 도메인 특화 데이터 과학에서 AI 에이전트가 인간 전문가 수준에 도달하는지는 불확실하다. AgentDS는 6개 산업(상업, 식품, 의료, 보험, 제조, 소매은행) 17개 챌린지로 구성된 벤치마크다. 29개 팀 80명 참가자의 오픈 대회를 통해 인간-AI 협업과 AI 전용 기준선을 비교했다. 결과: AI 전용 기준선은 중간 수준에 근접하거나 미달하며, 최강 솔루션은 인간-AI 협업에서 나왔다. 현재 AI 에이전트는 도메인 특화 추론에 취약하고 완전 자동화 서사에 도전장을 던지며 인간 전문성의 지속적 중요성을 강조한다.

### 핵심 기여
- 도메인 특화 데이터 과학에서 인간-AI 협업 평가를 위한 최초 대규모 벤치마크 구축
- AI 전용 기준선이 최고 인간 수준에 미달함을 경험적 규명
- 인간 전문성이 여전히 차별화 경쟁력임을 실증

`#DataScience #AIAgentBenchmark #HumanAICollaboration #Evaluation`

**DOI:** [10.48550/arXiv.2603.21321](https://doi.org/10.48550/arXiv.2603.21321)
**링크:** <https://arxiv.org/abs/2603.21321>

### 요약
고성능 시스템 휴리스틱 설계는 전문가의 가설 형성과 다단계 개념 전환이 필요한 창의적 반복 과정이다. LLM은 이 루프 자동화에 유망하나, 진화적 이웃 편향과 일관성 천장 두 가지 실패 모드에 직면한다. Engram은 장기 탐색을 단일 컨텍스트 윈도우 제약에서 분리하는 에이전트 연구자 아키텍처를 제안한다. 에이전트 시퀀스로 탐색을 조직하여 설계-테스트-분석을 반복하고, 각 실행 종료 시 코드 스냅샷·로그·결과를 영구 Archive에 저장하며 모델링 통찰을 Research Digest로 증류한다. 후속 에이전트는 새 컨텍스트로 시작하면서 Digest를 읽어 이전 발견을 기반으로 구축한다. ADRS 벤치마크 9개 문제 중 8개에서 인간 SOTA를 초과.

### 핵심 기여
- 에이전트 일관성 천장 문제를 Archive-Digest 패턴으로 해결
- 단일 컨텍스트 윈도우 제약을 벗어나는 지식 지속성 달성
- 다중 도메인(멀티클라우드, LLM 라우팅, KV 캐시 재사용)에서 효과 입증

**DOI:** [10.48550/arXiv.2603.19896](https://doi.org/10.48550/arXiv.2603.19896)
**링크:** <https://arxiv.org/abs/2603.19896>

### 요약
도구 사용 LLM 에이전트는 답변 품질과 실행 비용 간 근본적 긴장에 직면한다. 고정 워크플로우는 안정적이나 유연성이 부족하고, ReAct 등 자유형 방식은 성능 향상을 위해 과도한 도구 호출·토큰 소모·지연을 초래한다. 본 연구는 에이전트 오케스트레이션을 명시적 의사결정 문제로 다룬다. 효용 기반 오케스트레이션 정책이 respond, retrieve, tool call, verify, stop 등 액션을 예상 이득, 단계 비용, 불확실성, 중복성의 균형으로 선택한다. 다양한 조건(직접 답변, 임계값 제어, 고정 워크플로우, ReAct)에서 명시적 오케스트레이션 신호가 에이전트 행동에 실질적 영향을 미침을 실험으로 입증했다.

- 품질-비용 트레이드오프 분석을 위한 경량 효용 설계 제안
- 중복 제어와 워크플로우 공정성 분석 포함

### 요약
소규모 조직의 사이버보험 리스크 평가는 비용($15,000 이상)과 시간(수주)이 크고 전문가 부족으로 대부분 건너뛴다. 본 연구는 6개 에이전트(조직 프로파일링, 자산 매핑, 위협 분석, 통제 평가, 리스크 점수화, 권고 생성)로 구성된 다중 에이전트 AI 시스템을 구축했다. 에이전트는 지속적 컨텍스트를 공유하며, 후행 에이전트가 선행 결론을 기반으로 구축한다. 15인 HIPAA 의료 기업에서 CISSP 3인 평가와 비교하여 심각도 분류 85% 일치, 92% 리스크 포괄, 15분 이내 완료. ## 7. A Framework for Formalizing LLM Agent Security

**저자:** Vincent Siu et al.
**게재일:** 2026-03-19 | **분류:** cs.CR, cs.AI
**DOI:** [10.48550/arXiv.2603.19469](https://doi.org/10.48550/arXiv.2603.19469)
**링크:** <https://arxiv.org/abs/2603.19469>

### 요약
LLM 에이전트의 보안은 본질적으로 맥락적이다. 동일 액션이 지시 출처, 목표, 정당성에 따라 합법적 행동 또는 보안 위반이 될 수 있다. 기존 공격 정의는 이 맥락적 특성을 포착하지 못하여 방어 시 유틸리티-보안 트레이드오프 문제가 발생한다. 본 연구는 4가지 맥락적 보안 속성을 제안: task alignment(인가 목표 추구), action alignment(개별 액션이 목표에 기여), source authorization(인증 소스에서 명령 실행), data isolation(정보 흐름이 권한 경계 존중). 오라클 함수를 도입해 속성 위반 검증을 가능하게 하고, 간접 프롬프트 주입, 직접 프롬프트 주입, jailbreak, task drift, memory poisoning 등 기존 공격을 속성 위반으로 재정형화했다.

### 핵심 기여
- LLM 에이전트 보안의 맥락적 특성을 4가지 형식 속성으로 체계화
- 기존 공격/방어 분류를 통합 재정형화
- 유틸리티-보안 트레이드오프의 구조적 원인 규명

`#AgentSecurity #PromptInjection #FormalMethods #LLMSafety`

## 8. Performance-oriented Context Compression for LLMs via Performance Prediction (PoC)

**저자:** Runsong Zhao, Shilei Liu, Jiwei Tang et al.
**게재일:** 2026-03-20 | **분류:** cs.CL
**DOI:** [10.48550/arXiv.2603.19733](https://doi.org/10.48550/arXiv.2603.19733)
**링크:** <https://arxiv.org/abs/2603.19733>

### 요약
컨텍스트 압축은 LLM 추론 비용을 줄이지만, 기존 방법은 목표 압축 비율을 지정하면 성능 저하를 예측할 수 없어 신뢰성 있는 배포를 방해한다. PoC(Performance-oriented Context Compression)는 개발자가 압축 비율 대신 허용 가능한 성능 하한을 지정하는 패러다임 전환을 제안한다. 경량 성능 예측기가 이 제약을 만족하는 최대 압축 비율을 자동 탐색한 뒤 기존 압축기를 조절한다. 컨텍스트 비인지/인지 두 가지 예측기 변형을 설계·비교했고, QA와 요약 벤치마크에서 인지 예측기가 일관되게 낮은 예측 오차를 달성했다.

### 핵심 기여
- 압축 비율 기반에서 성능 보장 기반으로의 패러다임 전환
- 컨텍스트 고유 압축성을 고려한 인지 예측기 설계
- 성능 예측 오차 최소화와 효율적 압축 달성

## 9. PRISM: Breaking the O(n) Memory Wall via O(1) Photonic Block Selection

**저자:** Hyoseok Park et al.
**게재일:** 2026-03-23 | **분류:** physics.optics, cs.AI, cs.AR, cs.CL, cs.LG
**DOI:** [10.48550/arXiv.2603.21576](https://doi.org/10.48550/arXiv.2603.21576)
**링크:** <https://arxiv.org/abs/2603.21576>

### 요약
롱컨텍스트 LLM 추론의 병목은 연산이 아니라 모든 디코딩 스텝에서 KV 캐시를 스캔하는 O(n) 메모리 대역폭 비용이다. 기존 광학 가속기도 동일한 O(n) 메모리 스케일링을 상속받는다. 본 연구는 조잡 블록 선택 단계(메모리 바운드 유사도 검색)가 광학 broadcast-and-weight 패러다임과 구조적으로 일치함을 최초로 식별했다. 쿼리가 패시브 분할로 모든 후보에 확산되고, 서명은 준정적(전기광학 MRR 프로그래밍 일치), 순서만 중요(4-6비트 정도화). PRISM(TFLN 유사도 엔진)은 Qwen2.5-7B에서 4K~64K 토큰 100% 정확도(k=32), 64K 컨텍스트에서 16x 트래픽 감소, GPU 대비 4자리 에너지 우위 달성.

### 핵심 기여
- 광학 유사도 엔진으로 롱컨텍스트 KV 캐시 선택에서 O(1) 달성
- 컨텍스트 길이 증가에 따른 광학 이점 증가 수학적 입증
- 하드웨어 손상 평가에서 실질적 성능 검증

`#PhotonicComputing #LongContext #KVCache #HardwareArchitecture`
