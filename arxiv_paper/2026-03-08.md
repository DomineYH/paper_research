### 1) Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned
- 링크: https://arxiv.org/abs/2603.05344
- DOI: 10.48550/arXiv.2603.05344
- 핵심기여: 터미널 환경용 코딩 에이전트(OPENDEV) 설계에서 **안전한 실행 하네스·컨텍스트 계층화·장기작업 오케스트레이션**을 통합한 실무 프레임 제시.
- 요약(약 500자): 본 논문은 IDE 플러그인 중심 보조에서 CLI 중심 자율 에이전트로의 전환을 다룬다. 저자들은 소스 제어·빌드·배포가 실제로 일어나는 터미널을 에이전트의 1차 실행면으로 보고, 이를 위해 권한 제어, 명령 샌드박싱, 실패 복구 루프를 포함한 하네스 구조를 제안한다. 특히 컨텍스트 엔지니어링을 정적 프롬프트가 아닌 작업 히스토리/정책/저장소 규약의 동적 조합으로 다루며, 장기 과제에서 재시도 비용과 오류 전파를 줄이는 운영 원칙을 제시한다.
- 키워드: #AIAgent #ContextEngineering #CodingAgent #ToolReliability #TerminalOps

### 2) AegisUI: Behavioral Anomaly Detection for Structured User Interface Protocols in AI Agent Systems
- 링크: https://arxiv.org/abs/2603.05031
- DOI: 10.48550/arXiv.2603.05031
- 핵심기여: UI 스키마 유효성만으로 잡히지 않는 **행동 수준의 악성/오동작 패턴**을 탐지하는 보호 계층 제안.
- 요약(약 500자): 구조화된 UI 페이로드가 형식적으로는 정상이어도 실제 동작은 악의적일 수 있다는 문제를 다룬다. 예를 들어 버튼 라벨과 실제 액션의 의미가 어긋나거나 민감 필드가 은닉 바인딩되는 경우다. AegisUI는 컴포넌트 의미, 액션 체인, 데이터 흐름을 함께 검사해 스키마 검증 이후 단계의 이상행동을 잡도록 설계되었다. 이는 에이전트 기반 제품에서 ‘보이는 UI’와 ‘실행되는 행위’의 정합성을 검증하는 도구 신뢰성 프레임으로 활용 가치가 높다.
- 키워드: #ToolReliability #AISafety #UIProtocol #AgentSecurity #Evals

### 3) EVMbench: Evaluating AI Agents on Smart Contract Security
- 링크: https://arxiv.org/abs/2603.04915
- DOI: 10.48550/arXiv.2603.04915
- 핵심기여: 스마트컨트랙트 보안에서 에이전트의 **탐지·패치·익스플로잇** 능력을 동시 측정하는 평가 벤치마크 제시.
- 요약(약 500자): 블록체인 계약 코드가 고가치 자산을 직접 제어하는 상황에서, AI 에이전트의 보안 역량과 위험성을 함께 측정해야 한다는 관점의 벤치마크다. EVMbench는 취약점 식별 정확도뿐 아니라 패치의 기능 보존성, 공격 시나리오 재현 가능성까지 포함해 평가 범위를 확장한다. 단순 코드 정답률을 넘어 실제 운영환경 위험을 정량화하는 데 초점을 맞췄고, 향후 agent red-teaming과 안전성 인증 절차에 활용될 수 있는 기초 프레임을 제공한다.
- 키워드: #AIAgent #SecurityEvals #SmartContract #ToolUse #Reliability

### 4) Alignment Backfire: Language-Dependent Reversal of Safety Interventions Across 16 Languages in LLM Multi-Agent Systems
- 링크: https://arxiv.org/abs/2603.04904
- DOI: 10.48550/arXiv.2603.04904
- 핵심기여: 정렬 개입이 언어권에 따라 역효과를 낼 수 있음을 다국어 멀티에이전트 시뮬레이션으로 제시.
- 요약(약 500자): 본 연구는 안전 정렬 기법이 모든 언어에서 동일하게 작동한다는 가정을 반박한다. 16개 언어·다수 모델군의 멀티에이전트 실험에서 표면적 안전 응답이 실제 집단행동 개선으로 이어지지 않거나, 특정 언어권에서는 오히려 역전 효과가 나타날 수 있음을 보고한다. 이는 안전성 평가를 영어 중심 단일 벤치마크에 의존할 경우 과신 위험이 크다는 점을 시사한다. 실무적으로는 언어별 evals, 지역 맥락 테스트, 개입 전후 행동지표 추적의 필요성이 커진다.
- 키워드: #AISafety #MultiAgent #Evals #CrossLingual #Alignment

### 5) Jagarin: A Three-Layer Architecture for Hibernating Personal Duty Agents on Mobile
- 링크: https://arxiv.org/abs/2603.05069
- DOI: 10.48550/arXiv.2603.05069
- 핵심기여: 모바일 에이전트의 상시성-배터리 상충을 해결하기 위한 **하이버네이션 기반 3계층 아키텍처** 제안.
- 요약(약 500자): 모바일 개인 에이전트는 항상 켜두면 전력/OS 정책 제약에 걸리고, 완전 반응형이면 중요한 시점을 놓치는 딜레마가 있다. Jagarin은 긴급도 기반 깨움(DAWN), 제한적 백그라운드 판단, 필요 시만 활성화되는 실행층으로 역할을 분리한다. 이 구조는 지속 메모리와 의무(일정·알림) 추적을 유지하면서도 전력 비용을 최소화하도록 설계되었다. 에이전트 메모리 운용과 실서비스 배포 제약을 함께 고려했다는 점에서 개인 에이전트 운영 연구에 직접적 함의를 준다.
- 키워드: #AIAgent #AgentMemory #MobileAgent #ContextAwareness #Systems

### 6) Knowledge Divergence and the Value of Debate for Scalable Oversight
- 링크: https://arxiv.org/abs/2603.05293
- DOI: 10.48550/arXiv.2603.05293
- 핵심기여: debate 기반 감독의 이점을 모델 간 **지식 발산(knowledge divergence)** 기하로 정식화.
- 요약(약 500자): AI 안전의 확장 감독에서 debate와 RLAIF는 자주 병렬적으로 논의되지만, 언제 debate가 유리한지 이론적 조건은 부족했다. 이 논문은 모델 표현공간의 주각(principal angle)로 지식 발산을 측정하고, 발산 구조에 따라 debate 이득이 어떻게 변하는지 닫힌형 해를 제시한다. 즉, 서로 다른 오차를 가진 모델 조합에서는 상호 반박을 통해 감독 신호가 강화될 수 있고, 지나치게 유사한 모델끼리는 이득이 제한적이라는 점을 설명한다.
- 키워드: #AISafety #ScalableOversight #Debate #Evals #Theory

### 7) Censored LLMs as a Natural Testbed for Secret Knowledge Elicitation
- 링크: https://arxiv.org/abs/2603.05494
- DOI: 10.48550/arXiv.2603.05494
- 핵심기여: 인위적 거짓말 모델 대신 **검열된 실제 모델군**을 비밀지식 유도/거짓 탐지의 자연 실험장으로 제시.
- 요약(약 500자): 정직성 유도(honesty elicitation)와 거짓 탐지(lie detection) 연구는 종종 실험용으로 조작된 모델을 사용해 현실 일반화에 한계가 있었다. 본 논문은 검열 정책이 강한 공개모델을 대상으로, 민감지식 응답의 은닉·우회·회피 패턴을 측정한다. 이 접근은 실제 서비스 환경에 가까운 왜곡 메커니즘을 다룬다는 장점이 있으며, 안전 가드레일이 정보 왜곡으로 전이되는 경로를 분석할 수 있다. 단, 데이터 접근성과 정책 변화에 따라 재현성 관리가 필요하다.
- 키워드: #AISafety #Honesty #LieDetection #ModelBehavior #Evals

### 8) Diffusion LLMs can think EoS-by-EoS
- 링크: https://arxiv.org/abs/2603.05197
- DOI: 10.48550/arXiv.2603.05197
- 핵심기여: diffusion LLM의 추론 이득이 **EoS 토큰 구간의 내부 계산 과정**과 연관됨을 가설·실험으로 제시.
- 요약(약 500자): 확산형 LLM이 복합 추론 과제에서 보이는 강점의 원인으로, 저자들은 답을 낸 뒤의 EoS 구간이 단순 패딩이 아니라 추가적 내부 정련 단계로 기능할 수 있다고 본다. 생성 길이를 늘렸을 때 성능이 개선되는 현상을 분석해, 기존 자기회귀 모델과 다른 계산 동학을 제안한다. 이 관찰은 추론 비용/지연 최적화 전략, stop criterion 설계, 그리고 ‘생성 토큰 수=추론량’이라는 통념 재검토에 연결된다.
- 키워드: #LLM #Reasoning #DiffusionModel #Inference #Evals

### 9) SEA-TS: Self-Evolving Agent for Autonomous Code Generation of Time Series Forecasting Algorithms
- 링크: https://arxiv.org/abs/2603.04873
- DOI: 10.48550/arXiv.2603.04873
- 핵심기여: 시계열 예측 코드 생성에서 **자체 진화 루프(생성-검증-개선)**를 도입한 자율 에이전트 프레임 제안.
- 요약(약 500자): 새로운 도메인에서는 데이터 부족과 분포 이동으로 수동 모델 튜닝 효율이 급격히 떨어진다. SEA-TS는 에이전트가 후보 알고리즘 코드를 자동 생성하고, 지표 기반 검증 후 실패 유형을 반영해 다음 후보를 재작성하는 순환 구조를 채택한다. 이를 통해 인간이 반복하던 실험 설계 부담을 줄이고, 환경 변화에 대한 적응성을 높이는 것을 목표로 한다. 데이터사이언스 워크플로우 자동화 측면에서 AI agent + evaluation loop의 전형적 사례로 해석 가능하다.
- 키워드: #AIAgent #DataScience #AutoML #TimeSeries #SelfImprovement

### 10) Multimodal Adaptive Retrieval Augmented Generation through Internal Representation Learning
- 링크: https://arxiv.org/abs/2603.00511
- DOI: 10.48550/arXiv.2603.00511
- 핵심기여: 멀티모달 RAG에서 검색을 상시 고정하지 않고 **내부 표현 기반으로 동적 게이팅**하는 방법 제시.
- 요약(약 500자): 시각 질의응답에서 외부 검색은 환각을 줄이지만, 관련성 낮은 근거가 유입되면 오히려 성능을 악화시킬 수 있다. 저자들은 모델 내부 표현으로 불확실성을 추정해 ‘검색 사용/미사용’을 상황별로 전환하는 적응형 RAG를 제안한다. 핵심은 retrieval 품질이 낮을 때 검색 비중을 줄여 노이즈 전파를 억제하고, 불확실성이 큰 샘플에만 외부 지식을 적극 투입하는 것이다. 이는 정확도뿐 아니라 지연·비용 최적화 관점에서도 산업 적용성이 높다.
- 키워드: #MultimodalAgent #RAG #RAGEvaluation #ToolReliability #ContextEngineering
