### 1) AI Agent Systems: Architectures, Applications, and Evaluation
- 링크: https://arxiv.org/abs/2601.01743
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 에이전트형 AI 시스템을 아키텍처, 적용 도메인, 평가 방법으로 통합 정리한 최신 서베이 성격의 논문이다. 특히 장기 과업에서 발생하는 누적 오류, 비결정성, 도구 호출 실패, 재현성 저하를 핵심 병목으로 지적하며, 단순 태스크 성공률을 넘어 제약 하 성능·강건성·추적성 기반 평가 프레임이 필요하다고 주장한다. 메모리/RAG 결합 환경에서 컨텍스트 관리 품질이 성능 편차를 좌우한다는 점을 강조해, 프롬프트 중심 최적화에서 시스템 엔지니어링 중심 최적화로의 전환 필요성을 보여준다.
- 핵심 기여: 에이전트 시스템 평가를 “정확도 + 운영신뢰성” 다축으로 재정의.
- 키워드: #AIAgents #Evaluation #Reliability #ToolUse #AgentArchitecture

### 2) Agentic Artificial Intelligence (AI): Architectures, Taxonomies, and Evaluation of Large Language Model Agents
- 링크: https://arxiv.org/abs/2601.12560
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): LLM 에이전트 연구를 단일 모델 성능이 아닌 계획·기억·도구사용·권한제어를 포함한 운영 단위로 분류한 분류학 논문이다. 최근 추론형 모델 확산 이후, 고정 프롬프트 튜닝만으로는 장기 태스크의 일관성을 유지하기 어렵고, 상태관리 및 안전 제약을 포함한 오케스트레이션 계층이 필요하다는 점을 정리한다. 또한 멀티모달 상호작용과 구조화된 툴콜 환경에서 평가 항목을 세분화해야 함을 제시해 실무형 에이전트 개발 체크리스트로 활용 가능하다.
- 핵심 기여: LLM agent를 시스템 레벨 구성요소 중심으로 구조화한 평가 분류틀 제시.
- 키워드: #AgenticAI #LLMAgents #Taxonomy #SystemDesign #Evaluation

### 3) A Survey of Context Engineering for Large Language Models
- 링크: https://arxiv.org/abs/2507.13334
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 컨텍스트 엔지니어링을 프롬프트 작성의 하위 기법이 아니라, 모델 입력 생태계를 설계하는 독립 공학 영역으로 정리한 대규모 서베이다. 텍스트·도구 출력·검색 증거·멀티모달 신호를 어떻게 선별·압축·정렬할지에 따라 성능과 비용이 크게 달라짐을 다룬다. 특히 RAG 통합, 도메인 지식 주입, 장기 상호작용에서의 컨텍스트 일관성 문제가 반복적으로 등장하며, 단일 벤치마크 점수보다 운영 맥락에서의 안정성 측정이 중요하다는 함의를 제공한다.
- 핵심 기여: 컨텍스트 설계를 모델 외부의 핵심 최적화 변수로 체계화.
- 키워드: #ContextEngineering #Prompting #RAG #LLMSystems #Multimodal

### 4) Memory in the Age of AI Agents
- 링크: https://arxiv.org/abs/2512.13564
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 에이전트 메모리 연구를 형태(파라메트릭/비파라메트릭), 기능(회상/계획지원/자기반성), 동학(갱신/망각/통합)으로 통합한 최신 서베이다. 기존 연구가 저장 방식 개선에 치중한 반면, 실제 성능 병목은 검색/활용 단계에서 발생하는 경우가 많다는 최근 경향을 정리한다. 또한 멀티에이전트 협업에서 메모리 일관성과 접근제어가 안전성 이슈로 연결된다는 점을 제시해, 신뢰가능한 메모리 거버넌스 연구 필요성을 강조한다.
- 핵심 기여: 분절된 agent memory 연구를 공통 용어와 과제로 통합.
- 키워드: #AgentMemory #LongTermMemory #Survey #Trustworthiness #MultiAgent

### 5) Agentic Context Engineering: Evolving Contexts for Self-Improving Language Models
- 링크: https://arxiv.org/abs/2510.04618
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): 본 논문은 모델 파라미터를 바꾸지 않고도 컨텍스트를 동적으로 진화시켜 에이전트 성능을 개선할 수 있다는 관점을 제시한다. 정적 시스템 프롬프트 대신 실행 로그, 검색 증거, 실패 패턴을 반영해 컨텍스트 정책을 업데이트하는 루프를 제안하며, 정확도뿐 아니라 비용/지연/확장성 지표를 함께 고려해야 함을 강조한다. 특히 brevity bias와 context collapse 같은 실무적 실패 모드를 지적해, 장기 운영형 에이전트에서의 컨텍스트 품질관리 필요성을 구체화한다.
- 핵심 기여: “자가개선”을 가중치 학습이 아닌 컨텍스트 정책 진화로 재정의.
- 키워드: #AgenticContext #SelfImprovement #ContextPolicy #Reliability #LLM

### 6) A-Mem: Agentic Memory for LLM Agents
- 링크: https://arxiv.org/abs/2502.12110
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): A-Mem은 단순 저장소 기반 메모리를 넘어, 상황 적합성을 고려한 메모리 생성/검색/사용 전략을 통합적으로 설계한다. 특히 장기 대화/과업 환경에서 중요한 사실을 유지하면서 노이즈 기억을 억제하는 메커니즘을 제안해, 컨텍스트 길이 증가에 따른 품질 저하를 완화하려는 접근이다. RAG와 결합될 때는 검색 품질과 메모리 품질의 상호작용이 성능을 좌우하며, 메모리 설계가 프롬프트 엔지니어링 효과를 증폭시키는 기반층이 될 수 있음을 보여준다.
- 핵심 기여: 에이전트 실행 맥락을 반영한 동적 메모리 운용 프레임 제안.
- 키워드: #AMem #AgentMemory #RAG #LongHorizon #LLMAgents

### 7) Mem0: Building Production-Ready AI Agents with Scalable Long-Term Memory
- 링크: https://arxiv.org/abs/2504.19413
- DOI: 10.48550/arXiv.2504.19413
- 요약(약 500자): Mem0는 프로덕션 환경에서 장기 메모리를 안정적으로 운영하기 위한 실용 프레임워크를 제시한다. 기억 추출, 엔터티/관계 구조화, 검색 최적화를 결합해 사용자-에이전트 상호작용의 개인화와 연속성을 높이는 데 초점을 둔다. 연구의 실무적 의미는 “더 긴 컨텍스트 창”보다 “더 좋은 메모리 인덱싱”이 비용 대비 효과가 크다는 점이며, 실제 서비스에서 메모리 누적에 따른 드리프트·편향 위험을 관리하는 운영 규칙의 중요성도 함께 제시한다.
- 핵심 기여: 장기 메모리의 서비스 운영 관점(확장성·일관성) 구체화.
- 키워드: #Mem0 #ProductionAgents #LongTermMemory #Personalization #Scalability

### 8) Agentic Retrieval-Augmented Generation: A Survey on Agentic RAG
- 링크: https://arxiv.org/abs/2501.09136
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): Agentic RAG 연구를 검색-추론-행동 루프 관점에서 정리한 서베이다. 기존 RAG가 정적 검색 후 단발 생성에 가깝다면, 에이전트형 RAG는 질의 재구성, 증거 재탐색, 자기검증을 반복해 품질을 높인다는 점을 강조한다. 다만 루프 구조가 길어질수록 지연시간·비용·오류 전파 위험이 커져, 평가에서 정확도만이 아니라 효율성과 안전성까지 동시에 측정해야 한다는 과제를 제시한다. 이는 기업형 지식검색 시스템의 설계 기준으로 직접 연결된다.
- 핵심 기여: Agentic RAG 평가를 다목적 최적화 문제로 정식화.
- 키워드: #AgenticRAG #Retrieval #ToolUse #Evaluation #KnowledgeSystems

### 9) COMMA: A Communicative Multimodal Multi-Agent Benchmark
- 링크: https://arxiv.org/abs/2410.07553
- DOI: 10.48550/arXiv.2410.07553
- 요약(약 500자): COMMA는 멀티모달 입력과 에이전트 간 커뮤니케이션을 동시에 요구하는 벤치마크를 제안한다. 단일 에이전트 정확도보다, 협업 과정에서 정보 전달 손실·오해·역할분담 실패가 성능을 좌우한다는 점을 실험적으로 보여준다. 이는 향후 멀티에이전트 시스템에서 메시지 프로토콜, 공유 메모리, 설명가능 추적 로그가 왜 필요한지를 뒷받침하며, 실제 업무 자동화에서 빈번한 “팀 단위 에이전트 실패”를 정량화할 기반을 제공한다.
- 핵심 기여: 멀티모달+협업 커뮤니케이션을 결합한 멀티에이전트 평가틀 제공.
- 키워드: #COMMA #MultimodalAgents #MultiAgent #Benchmark #Communication

### 10) MMDeepResearch-Bench: A Benchmark for Multimodal Deep Research Agents
- 링크: https://arxiv.org/abs/2601.12346
- DOI: 확인 불가(사전공개본)
- 요약(약 500자): MMDeepResearch-Bench는 멀티모달 심층조사 에이전트의 엔드투엔드 능력을 측정하도록 설계된 벤치마크다. 텍스트·이미지·웹 단서를 통합해 문제를 해결해야 하므로, 단순 QA 성능과 달리 증거 수집 전략·출처 정합성·추론 체인 안정성을 함께 평가한다. 특히 보고서형 출력 품질과 근거 추적성을 함께 점검한다는 점에서, 연구 자동화 파이프라인(문헌탐색→요약→주제도출) 검증에 적합하다.
- 핵심 기여: Deep-research agent의 실사용 시나리오에 가까운 멀티모달 평가 환경 제시.
- 키워드: #DeepResearch #Multimodal #Benchmark #EvidenceTracing #AgentEvaluation
