## 1. From Storage to Steering: Memory Control Flow Attacks on LLM Agents

- **저자**: Zhenlin Xu et al.
- **링크**: <https://arxiv.org/abs/2603.15125>
- **DOI**: 10.48550/arXiv.2603.15125
- **카테고리**: cs.CR
- **등록일**: 2026-03-16

**요약**: 현대 에이전트 시스템은 LLM이 도구를 반복적으로 호출하며 구조화된 제어 흐름을 형성하지만, 기존 보안 분석은 메모리의 지속적 영향을 간과해왔다. 본 논문은 메모리 검색이 제어 흐름을 지배하여 명시적 사용자 지시에도 불구하고 의도치 않은 도구 사용을 강제하고 작업 간 지속적 행동 편차를 유발하는 Memory Control Flow Attacks(MCFA)를 식별한다. MEMFLOW 자동 평가 프레임워크를 설계하여 GPT-5 mini, Claude Sonnet 4.5, Gemini 2.5 Flash를 LangChain 및 LlamaIndex 프레임워크에서 평가한 결과, 엄격한 안전 제약 하에서도 90% 이상의 시도가 MCFA에 취약한 것으로 나타났다.

**핵심 기여**: 에이전트 메모리 시스템의 제어 흐름 공격(MCFA)이라는 새로운 위협 모델 제시 및 MEMFLOW 평가 프레임워크 개발

**키워드**: #AgentMemory #MemoryAttack #ControlFlow #LLMSecurity #LangChain #LlamaIndex

## 2. CLAG: Adaptive Memory Organization via Agent-Driven Clustering for Small Language Model Agents

- **저자**: Taeyun Roh et al.
- **링크**: <https://arxiv.org/abs/2603.15421>
- **DOI**: 10.48550/arXiv.2603.15421
- **카테고리**: cs.CL, cs.AI
- **등록일**: 2026-03-16

**요약**: 소형 언어 모델(SLM) 에이전트는 단일 전역 검색 풀에 경험을 저장하는 기존 메모리 시스템의 한계로 인해 지식 희석 및 손상에 매우 취약하다. CLAG는 SLM 에이전트가 능동적으로 클러스터링으로 메모리를 구성하는 프레임워크로, SLM 기반 라우터가 수신 메모리를 의미론적으로 일관된 클러스터에 할당하고 클러스터별 프로필(주제 요약, 서술적 태그)을 자율 생성한다. 검색 시 2단계 프로세스(프로필 기반 필터링 → 국소 검색)로 교차 주제 간섭을 효과적으로 감소시킨다. 다중 QA 데이터셋에서 기존 메모리 시스템 대비 일관된 성능 향상을 달성했다.

**핵심 기여**: SLM 에이전트를 위한 경량 클러스터 기반 적응형 메모리 구성 프레임워크로, 교차 주제 간섭 감소 및 메모리 밀도 향상

**키워드**: #SmallLanguageModel #AgentMemory #Clustering #AdaptiveRetrieval #MemoryOrganization

## 3. Evaluating Memory in LLM Agents via Incremental Multi-Turn Interactions (MemoryAgentBench)

- **저자**: Yuanzhe Hu, Yuqi Wang et al. (동등 기여)
- **링크**: <https://arxiv.org/abs/2507.05257>
- **DOI**: 10.48550/arXiv.2507.05257
- **카테고리**: cs.CL, cs.AI
- **최신 업데이트**: 2026-03-17 (v3)

**요약**: 기존 LLM 에이전트 벤치마크는 추론, 계획, 실행에 집중하며 메모리(기억, 갱신, 검색) 평가는 부족하다. 인지과학 이론에 기반하여 정확한 검색(accurate retrieval), 테스트 시간 학습(test-time learning), 장기 이해(long-range understanding), 선택적 망각(selective forgetting)의 4가지 핵심 역량을 도출하고, MemoryAgentBench를 제안한다. 다중 턴 증분 정보 처리 형식으로 기존 장문맥 데이터셋을 변환하여 에이전트의 대화적 특성을 반영한다. 단순 컨텍스트 기반 시스템부터 RAG, 외부 메모리 모듈 탑재 에이전트까지 평가한 결과, 어떤 방법도 4가지 역량을 모두 완벽히 달성하지 못함을 확인했다.

**핵심 기여**: 메모리 에이전트를 위한 4역량 체계적 벤치마크 MemoryAgentBench 제안 및 기존 방법의 한계 실증

**키워드**: #MemoryAgentBench #MemoryEvaluation #IncrementalLearning #RAG #LongTermMemory

## 4. Memento-Skills: Let Agents Design Agents

- **저자**: Huichi Zhou, Siyuan Guo, Anjie Liu et al.
- **링크**: <https://arxiv.org/abs/2603.18743>
- **카테고리**: cs.AI, cs.CL, cs.LG
- **등록일**: 2026-03-19

**요약**: Memento-Skills는 메모리 기반 강화학습 프레임워크를 기반으로 한 범용 지속학습 에이전트 시스템으로, "에이전트를 설계하는 에이전트(agent-designing agent)"로 기능한다. 재사용 가능한 스킬을 구조화된 마크다운 파일로 저장하여 지속적 진화하는 메모리로 활용하며, Read-Write Reflective Learning 메커니즘을 통해 행동 가능한 스킬 라우터가 관련 스킬을 선택(Read)하고 새로운 경험으로 스킬 라이브러리를 갱신(Write)한다. LLM 파라미터 업데이트 없이 외부화된 스킬과 프롬프트의 진화만으로 지속 학습을 실현하며, General AI Assistants 벤치마크에서 26.2%, Humanity's Last Exam에서 116.2%의 상대적 정확도 향상을 달성했다.

**핵심 기여**: 파라미터 업데이트 없이 외부 스킬 진화를 통한 지속 학습 구현 및 에이전트 자율 설계 패러다임 제시

**키워드**: #AgentDesign #ContinualLearning #SkillMemory #ReflectiveLearning #NoParamUpdate

## 5. MemMA: Coordinating the Memory Cycle through Multi-Agent Reasoning and In-Situ Self-Evolution

- **저자**: Minhua Lin et al.
- **링크**: <https://arxiv.org/abs/2603.18718>
- **DOI**: 10.48550/arXiv.2603.18718
- **카테고리**: cs.AI
- **등록일**: 2026-03-19

**요약**: 기존 메모리 증강 LLM 에이전트는 메모리의 구성, 검색, 활용을 독립된 서브루틴으로 처리하여 전략적 맹점(construct/retrieval이 국소 휴리스틱에 의존)과 희소·지연된 감독(하류 실패가 메모리 은행 직접 수정으로 이어지지 않음)이라는 두 가지 문제를 야기한다. MemMA는 순방향 경로에서 Meta-Thinker가 구조화된 지침을 생성하여 Memory Manager와 Query Reasoner를 조율하고, 역방향 경로에서 probe QA 합성, 메모리 검증, 실패→수정 액션 변환을 통한 제자리 자가 진화(in-situ self-evolution)를 도입한 플러그앤플레이 다중 에이전트 프레임워크이다. LoCoMo에서 다중 LLM 백본 및 스토리지 백엔드에 대해 일관된 성능 향상을 달성했다.

**핵심 기여**: 메모리 사이클의 순/역방향 통합 조율 및 제자리 자가 진화를 통한 메모리 품질 향상

**키워드**: #MemMA #MultiAgent #MemoryCycle #SelfEvolution #PlugAndPlay

## 6. Measuring and Exploiting Confirmation Bias in LLM-Assisted Security Code Review

- **저자**: Dimitris Mitropoulos, Nikolaos Alexopoulos, Georgios Alexopoulos, Diomidis Spinellis
- **링크**: <https://arxiv.org/abs/2603.18740>
- **카테고리**: cs.SE, cs.AI, cs.CR
- **등록일**: 2026-03-19

**요약**: LLM 기반 보안 코드 리뷰에서 확인 편향(confirmation bias)이 취약점 탐지에 미치는 영향을 정량화하고 이를 공급망 공격에 악용할 수 있음을 보여준다. Study 1에서 250개 CVE 취약점/패치 쌍을 4개 SOTA 모델에 5가지 프레이밍 조건으로 평가한 결과, "버그 없음"으로 프레이밍하면 취약점 탐지율이 16~93% 감소하며 위음성이 급증한다. Study 2에서 GitHub Copilot에 대해 35%, Claude Code에 대해 88%의 공격 성공률을 기록했다. 메타데이터 제거 및 명시적 디바이싱 명령으로 인터랙티브 경우 100%, 자율 경우 94%에서 탐지 복구가 가능하다.

**핵심 기여**: LLM 코드 리뷰의 확인 편향 정량화, 실제 공급망 공격 시나리오에서의 악용 가능성 실증, 디바이싱 방안 제시

**키워드**: #ConfirmationBias #CodeReview #LLMSecurity #SupplyChainAttack #Debiasing

## 7. Nemotron-Cascade 2: Post-Training LLMs with Cascade RL and Multi-Domain On-Policy Distillation

- **저자**: Zhuolin Yang, Zihan Liu, Yang Chen et al. (NVIDIA)
- **링크**: <https://arxiv.org/abs/2603.19220>
- **카테고리**: cs.CL, cs.AI, cs.LG
- **등록일**: 2026-03-19

**요약**: NVIDIA의 Nemotron-Cascade 2는 3B 활성 파라미터의 오픈 30B MoE 모델로, 프론티어 오픈 모델에 근접한 수학·코딩 추론 능력을 갖춘다. DeepSeekV3.2-Speciale-671B-A37B에 이어 IMO, IOI, ICPC World Finals에서 Gold Medal 수준 성능을 달성한 두 번째 오픈 웨이트 LLM으로, 20배 적은 파라미터로 최고 수준의 지능 밀도(intelligence density)를 보여준다. 핵심 기술은 SFT 후 Cascade RL을 추론 및 에이전트 도메인으로 대폭 확장하고, 다중 도메인 온폴리시 증류(multi-domain on-policy distillation)로 중간 교사 모델로부터 효율적으로 벤치마크 회귀를 복구하는 것이다.

**핵심 기여**: 20배 파라미터 효율로 Gold Medal 수준 추론 달성, Cascade RL 확장 및 다중 도메인 온폴리시 증류 기법

**키워드**: #Nemotron #MoE #CascadeRL #OnPolicyDistillation #AgenticCapabilities

## 8. Box Maze: A Process-Control Architecture for Reliable LLM Reasoning

- **저자**: Zou Qiang
- **링크**: <https://arxiv.org/abs/2603.19182>
- **카테고리**: cs.AI, cs.CL
- **등록일**: 2026-03-19

**요약**: 기존 RLHF 및 출력 필터링 등 안전 접근법은 행동 수준에서 작동하며, 추론 프로세스 무결성을 보장하는 명시적 아키텍처 메커니즘이 부족하다. Box Maze는 LLM 추론을 메모리 접지(memory grounding), 구조화 추론(structured inference), 경계 강제(boundary enforcement)의 3개 명시적 계층으로 분해하는 프로세스 제어 아키텍처이다. DeepSeek-V3, Doubao, Qwen을 대상으로 n=50 적대적 시나리오에서 점진적 경계 침식 평가를 수행한 결과, 아키텍처 제약으로 경계 실패율을 RLHF 베이스라인 약 40%에서 1% 미만으로 감소시켰다.

**핵심 기여**: 추론 프로세스의 3계층 분해 아키텍처로 적대적 프롬프트 하에서 경계 실패율을 극적으로 감소

**키워드**: #ProcessControl #ReliableReasoning #BoundaryEnforcement #AdversarialPrompting #ArchitectureSafety

## 9. DaPT: A Dual-Path Framework for Multilingual Multi-hop Question Answering

- **저자**: Yilin Wang, Yuchun Fan, Jiaoyang Li et al.
- **링크**: <https://arxiv.org/abs/2603.19097>
- **카테고리**: cs.CL, cs.AI
- **등록일**: 2026-03-19 (ICASSP 2026 수록)

**요약**: 다국어 말뭉치와 쿼리를 검색해야 하는 RAG 시스템의 다국어 다중 홉(multi-hop) QA 설정에서 성능 불균형 문제를 다룬다. 영어 전용 벤치마크를 5개 언어로 번역하여 다국어 다중 홉 QA 벤치마크를 구성하고, DaPT 프레임워크를 제안한다. DaPT는 원어 쿼리와 영어 번역본에 대해 병렬로 하위 질문 그래프를 생성한 후 병합하고, 이중언어 검색-응답 전략으로 하위 질문을 순차적으로 해결한다. MuSiQue 벤치마크에서 최강 베이스라인 대비 평균 EM 점수 18.3%의 상대적 향상을 달성했다.

**핵심 기여**: 다국어 RAG 시스템의 성능 불균형 문제 해결을 위한 이중 경로 프레임워크 및 다국어 다중 홉 QA 벤치마크 구축

**키워드**: #MultilingualRAG #MultiHopQA #DualPath #ICASSP2026 #CrossLingualRetrieval

## 10. Mi:dm K 2.5 Pro

- **저자**: KT Tech Innovation Group
- **링크**: <https://arxiv.org/abs/2603.18788>
- **카테고리**: cs.CL, cs.AI
- **등록일**: 2026-03-19

**요약**: KT에서 개발한 32B 파라미터 한국어 특화 엔터프라이즈 LLM로, 다단계 추론, 장문맥 이해, 에이전트 워크플로우에 최적화되었다. Depth Upscaling(DuS) 기반 레이어 예측자로 사전 학습을 확장하고 128K 토큰 컨텍스트 윈도우를 지원한다. 사후 학습에서 Reasoning SFT, 모델 병합, 비동기 RL, Fusion Training을 통해 복잡 문제 해결 능력과 대화 유창성·도구 사용 신뢰성을 균형 있게 확보한다. 한국어 특화 벤치마크에서 SOTA를 달성하며 Responsible AI 평가로 안전성도 검증했다. 국내 모델의 글로벌 경쟁력을 입증하는 의미 있는 결과이다.

**핵심 기여**: 한국어 특화 32B 엔터프라이즈 LLM으로, Reasoning SFT+RL 파이프라인과 128K 컨텍스트 지원으로 글로벌 모델 대비 경쟁력 입증

**키워드**: #KoreanLLM #EnterpriseAI #DepthUpscaling #ReasoningSFT #KT #128KContext
