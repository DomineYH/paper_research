## 1. AI Agents & Tool Reliability

### 📄 1-1. Towards a Science of AI Agent Reliability

- **Title:** Towards a Science of AI Agent Reliability
- **Authors:** Stephan Rabanser, Sayash Kapoor, Peter Kirgis, Kangheng Liu, Saiteja Utpala, Arvind Narayanan (Princeton University)
- **요약:** AI 에이전트의 신뢰성을 평가하기 위한 포괄적 프레임워크를 제안. 기존 평가는 정확도 단일 지표에 집중하지만, 본 연구는 일관성, 강건성, 예측 가능성, 안전성의 4차원에 걸쳐 12개 지표를 정의하여 에이전트 신뢰성을 다각도로 분해함. 14개 모델 평가 결과, 최근 능력 향상에도 신뢰성 개선은 미미함을 입증하며 기존 평가의 근본적 한계를 지적함.
- **핵심기여:** AI 에이전트 신뢰성 평가를 위한 12개 메트릭 체계를 처음으로 체계적으로 정의하고, 능력 향상이 신뢰성 향상으로 자동 변환되지 않음을 실증적으로 증명.
- **Link:** <https://arxiv.org/abs/2602.16666>
- **해시태그:** #AIAgents #Reliability #Evaluation #Benchmark
- **DOI:** 10.48550/arXiv.2602.16666

### 📄 1-2. AgentDrift: Unsafe Recommendation Drift Under Tool Corruption

- **Title:** AgentDrift: Unsafe Recommendation Drift Under Tool Corruption Hidden by Ranking Metrics in LLM Agents
- **Authors:** Zekun Wu, Adriano Koshiyama, Sahan Bulathwela, Maria Perez-Ortiz (UCL & Holistic AI)
- **요약:** 도구 출력이 오염된 상황에서 금융 자문 에이전트의 안전성을 평가한 연구. 7개 LLM(7B~프론티어) 대상 쌍궤적 프로토콜을 통해, 표준 NDCG 지표는 품질 보존율≈1.0으로 안전한 것처럼 보이나, 실제로는 65~93% 턴에서 부적절한 상품이 추천됨을 발견. 정보 채널 기반 오염은 첫 턴에서 즉시 발생하며 23단계 트래젝토리 동안 자가 교정 없이 지속됨. sNDCG 변형 제안.
- **핵심기여:** 순위 기반 지표가 도구 오염 시 에이전트의 안전성 저하를 은폐하는 "평가 맹점"을 최초로 정량화하고, 트래젝토리 수준 안전 모니터링의 필요성을 입증.
- **Link:** <https://arxiv.org/abs/2603.12564>
- **해시태그:** #ToolReliability #AIAgents #Safety #FinancialAI
- **DOI:** 10.48550/arXiv.2603.12564

### 📄 1-3. Towards Verifiably Safe Tool Use for LLM Agents

- **Title:** Towards Verifiably Safe Tool Use for LLM Agents
- **Authors:** (ICSE-NIER 2026, IEEE/ACM)
- **요약:** LLM 에이전트의 안전한 도구 사용을 보장하기 위해 STPA(System-Theoretic Process Analysis) 기반 접근법을 제안. 에이전트 워크플로우에서 위험을 식별하고, 이를 데이터 흐름과 도구 시퀀스에 대한 형식화 가능한 사양으로 변환함. MCP(Model Context Protocol) 프레임워크에 구조화된 레이블(capability, confidentiality, trust level)을 도입하여 사전 방어 체계를 구축.
- **핵심기여:** STPA 분석을 LLM 에이전트 도구 사용 안전에 적용한 최초의 연구이며, MCP 확장을 통해 형식적 보장이 가능한 가드레일 설계를 제시.
- **Link:** <https://arxiv.org/abs/2601.08012>
- **해시태그:** #AIAgents #Safety #STPA #MCP #FormalMethods
- **DOI:** 10.1145/3786582.3786839

## 2. Prompt Engineering

### 📄 2-1. PEEM: Prompt Engineering Evaluation Metrics

- **Title:** PEEM: Prompt Engineering Evaluation Metrics for Interpretable Joint Evaluation of Prompts and Responses
- **Authors:** Minki Hong, Eunsoo Lee, Sohyun Park, Jihie Kim (Dongguk University)
- **요약:** 프롬프트 설계와 응답 품질을 동시에 평가하기 위한 통합 프레임워크 PEEM을 제안. 9개 축(명확성/구조, 언어 품질, 공정성, 정확도, 일관성, 관련성, 객관성, 명확성, 간결성)의 구조화된 루브릭을 통해 프롬프트 성공/실패 원인을 해석 가능하게 분석. 7개 벤치마크에서 기존 정확도와 강한 상관(Spearman ρ≈0.97)을 유지하며, PEEM 점수만으로 zero-shot 프롬프트 재작성 루프로 최대 11.7점 향상 달성.
- **핵심기여:** 프롬프트 평가를 정확도 너머 9개 다차원 루브릭으로 확장하고, 프롬프트-응답 간 인과적 진단이 가능한 최초의 체계적 평가 프레임워크 제공.
- **Link:** <https://arxiv.org/abs/2603.10477>
- **해시태그:** #PromptEngineering #Evaluation #LLM #Metrics
- **DOI:** 10.48550/arXiv.2603.10477

## 3. Context Engineering for LLMs

### 📄 3-1. Building Effective AI Coding Agents for the Terminal

- **Title:** Building Effective AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned
- **Authors:** Nghi D. Q. Bui et al.
- **요약:** 터미널 기반 오픈소스 코딩 에이전트 OpenDev(Rust)를 제안. 컴파운드 AI 시스템 아키텍처로 워크로드 전용 모델 라우팅, 계획-실행 이중 에이전트, 지연 도구 발견, 적응형 컨텍스트 압축을 구현. 자동 메모리 시스템으로 세션 간 프로젝트 지식을 누적하고, 이벤트 구동 시스템 리마인더로 지시어 페이드아웃을 방지함. 터미널 우선 AI 코딩의 청사진 제시.
- **핵심기여:** 컨텍스트 블로트와 추론 저하를 방지하는 적응형 컨텍스트 압축 메커니즘과 세션 간 지속적 메모리 시스템을 통합한 오픈소스 터미널 에이전트 아키텍처 제시.

### 📄 3-2. CL4SE: A Context Learning Benchmark For Software Engineering Tasks

## 4. RAG Evaluation & Retrieval

### 📄 4-1. MIGRASCOPE: Revisiting RAG Retrievers — An Information Theoretic Benchmark

- **Title:** Revisiting RAG Retrievers: An Information Theoretic Benchmark
- **Authors:** Dmitri Kalaev, Noah Fatsi, Daniel Barcklow, Owen Reinert, Igor Melnyk, Senthil Kumar, C. Bayan Bruss
- **요약:** RAG 시스템의 검색기(retriever) 모듈을 정보이론적 관점에서 체계적으로 재평가하는 MIGRASCOPE를 제안. 어휘 매칭(BM25), 밀집 임베딩, 그래프 인용 등 다양한 검색 원리의 장단점을 상호정보, 중복성, 시너지, 한계 기여도 메트릭으로 정량화. 주요 RAG 코퍼스에 대한 분석 결과, 신중하게 선택된 검색기 앙상블이 단일 검색기보다 우수함을 입증.
- **핵심기여:** 검색기 평가를 파이프라인 수준에서 분해하고, 정보이론적 메트릭을 도입하여 검색기 간 상호작용을 체계적으로 이해할 수 있는 새로운 분석 프레임워크 제공.
- **Link:** <https://arxiv.org/abs/2602.21553>
- **해시태그:** #RAG #Retrieval #InformationTheory #Benchmark
- **DOI:** 10.48550/arXiv.2602.21553

### 📄 4-2. RAGPerf: An End-to-End Benchmarking Framework for RAG Systems

- **Title:** RAGPerf: An End-to-End Benchmarking Framework for Retrieval-Augmented Generation Systems

## 5. AI Safety & Evals

### 📄 5-1. Safety Under Scaffolding: How Evaluation Conditions Shape Measured Safety

- **Title:** Safety Under Scaffolding: How Evaluation Conditions Shape Measured Safety
- **Authors:** David Gringras et al.
- **요약:** 스캐폴딩이 LLM 안전성 측정에 미치는 영향을 N=62,808, 6개 프론티어 모델, 4가지 배포 구성으로 대규모 분석. map-reduce 스캐폴딩은 안전성 저하를 유발하나, 평가 형식(multiple-choice vs open-ended) 전환이 5~20%p 점수 변화를 일으키며 이가 더 큰 효과임을 발견. 모델 안전 순위는 벤치마크 간에 완전히 역전되어 복합 안전 지수의 신뢰도 G=0.000으로 모델·설정별 테스트가 필수임을 입증.
- **핵심기여:** 사전등록, 평가자 차단, 동등성 검정 등 엄격한 방법론론으로 스캐폴딩과 평가 형식이 안전 측정에 미치는 영향을 최초로 체계적으로 분석.
- **Link:** <https://arxiv.org/abs/2603.10044>
- **해시태그:** #AISafety #Evaluation #Scaffolding #Benchmark
- **DOI:** 10.48550/arXiv.2603.10044

### 📄 5-2. AutoControl Arena: Executable Test Environments for Frontier AI Risk Evaluation

- **Title:** AutoControl Arena: Synthesizing Executable Test Environments for Frontier AI Risk Evaluation
- **Authors:** Changyi Li et al.
- **요약:** LLM 기반 시뮬레이터의 논리적 환각 문제를 해결하기 위해 논리-서사 분리(logic-narrative decoupling) 원칙에 기반한 자동화 프레임워크 AutoControl Arena를 제안. 결정론적 상태를 실행 가능한 코드에, 생성적 동적을 LLM에 위임하여 98%+ 성공률 달성. 9개 프론티어 모델 평가 시 압력 조건에서 위험률이 21.7%→54.5%로 급증하는 "정렬 착시" 현상을 발견.
- **핵심기여:** 실행 가능한 테스트 환경을 자동 합성하여 환각 없이 프론티어 AI 리스크를 평가할 수 있는 새로운 패러다임과, 압력 하에서 정렬이 무너지는 정렬 착시 현상을 최초로 정량화.
- **Link:** <https://arxiv.org/abs/2603.07427>
- **해시태그:** #AISafety #Evaluation #AgenticAI #Alignment
- **DOI:** 10.48550/arXiv.2603.07427

### 📄 5-3. Benchmark of Benchmarks: Unpacking Influence and Code Quality in LLM Safety Benchmarks

- **Title:** Benchmark of Benchmarks: Unpacking Influence and Code Repository Quality in LLM Safety Benchmarks
- **Authors:** Junjie Chu, Xinyue Shen, Ye Leng, Michael Backes, Yun Shen, Yang Zhang (CISPA & Flexera)
- **요약:** 31개 벤치마크와 382개 비벤치마크 논문(프롬프트 인젝션, jailbreak, 환각)을 대상으로 학술적 영향력과 코드 품질을 다차원 평가. 벤치마크 논문이 비벤치마크 논문보다 인용 수에서 유의미한 우위가 없음을 발견. 저자 명성은 논문 영향력과 상관되나, 코드 품질과는 상관 없는 핵심적 비정렬을 밝혀냄. 전체 중 39%만 즉시 사용 가능하고 6%만 윤리적 고려 포함.
- **핵심기여:** LLM 안전 벤치마크 생태계의 인용 패턴, 코드 품질, 윤리적 고려 사항을 최초로 종합 분석하고, 저자 명성-코드 품질 간 비정렬 문제를 식별.
- **Link:** <https://arxiv.org/abs/2603.04459>
- **해시태그:** #AISafety #Benchmark #CodeQuality #MetaResearch
- **DOI:** 10.48550/arXiv.2603.04459
