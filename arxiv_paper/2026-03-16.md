### 1) AI Agent Systems: Architectures, Applications, and Evaluation (arXiv:2601.01743)
- 링크: https://arxiv.org/abs/2601.01743
- DOI: https://doi.org/10.48550/arXiv.2601.01743
- 요약(약 500자): 본 서베이는 AI 에이전트를 추론·계획·메모리·도구호출의 결합 시스템으로 재정의하고, 단일/다중 에이전트, 중앙집중/분산 오케스트레이션, 오프라인/온라인 배치 등 배치 축을 통합 분류한다. 특히 지연시간-정확도, 자율성-통제가능성, 능력-신뢰성의 상충관계를 실무 설계의 핵심 변수로 제시하며, 장기 과제에서의 비결정성·재시도 비용·컨텍스트 팽창이 평가를 왜곡하는 문제를 다룬다. 벤치마크는 단일 정답률보다 제약하 성공률, 보안·강건성, 인간 선호를 함께 측정해야 한다는 점을 강조한다.
- 핵심 기여: 에이전트 시스템 설계·평가의 통합 택소노미 제안
- 키워드: #AIAgent #Evaluation #ToolUse #Memory #ContextEngineering

### 2) Context Engineering for Multi-Agent LLM Code Assistants (arXiv:2508.08322)
- 링크: https://arxiv.org/abs/2508.08322
- DOI: https://doi.org/10.48550/arXiv.2508.08322
- 요약: 다중 에이전트 코드 어시스턴트에서 성능 저하의 주원인을 "모델 능력 부족"이 아니라 "컨텍스트 조달 실패"로 보고, 의도 번역기·문헌 검색·문서 합성·코드 검증 에이전트를 계층 결합하는 워크플로를 제시한다. 단일 에이전트 대비 저장소 맥락 정합성과 원샷 성공률이 개선되었다고 보고하며, 실제 레포지토리에서 계획-수정-테스트 루프의 안정성이 높아졌다고 주장한다. 정량 벤치마크 일반화 범위는 제한적일 수 있어(불확실성), 재현 실험의 공개 범위가 후속 검증 포인트다.
- 핵심 기여: 코드 생성 파이프라인에서 컨텍스트 엔지니어링의 모듈화 실증
- 키워드: #ContextEngineering #MultiAgent #CodeAssistant #RAG #ToolReliability

### 3) A Survey of Context Engineering for LLMs (arXiv:2507.13334)
- 링크: https://arxiv.org/abs/2507.13334
- DOI: https://doi.org/10.48550/arXiv.2507.13334
- 요약: 1,400편 이상을 종합해 컨텍스트 엔지니어링을 검색·생성·처리·관리의 네 계층으로 구조화하고, 이를 RAG·메모리 시스템·도구통합 추론·멀티에이전트 구현으로 연결한다. 특히 현재 LLM이 "복잡한 컨텍스트 이해"에는 빠르게 진보했지만 "동등한 수준의 장문 생성"은 병목이라는 비대칭 문제를 제시한다. 이 관찰은 reverse prompt 분석, 장기 메모리 압축, 컨텍스트 비용 제어 연구의 필요성을 지지한다. 대규모 서베이 특성상 일부 하위영역 최신 결과 반영 시차 가능성은 존재한다(불확실성).
- 핵심 기여: 컨텍스트 엔지니어링의 표준 프레임과 연구 공백 명시
- 키워드: #ContextEngineering #LLM #AgentMemory #RAG #PromptEngineering

### 4) Reasoning Agentic RAG Survey (arXiv:2506.10408)
- 링크: https://arxiv.org/abs/2506.10408
- DOI: https://doi.org/10.48550/arXiv.2506.10408
- 요약: 정적 RAG의 한계를 해결하기 위해 Reasoning Agentic RAG를 사전정의 추론(System 1형 파이프라인)과 자율 추론(System 2형 에이전트 오케스트레이션)으로 구분하고, 검색-추론 결합 방식을 비교한다. 핵심은 검색을 단순 전처리가 아니라 의사결정 과정으로 끌어올리는 점이며, 산업 맥락에서 강건성·도구 실패 복구·멀티모달 결합의 난제를 제시한다. 실무적으로는 "검색 전략 선택"과 "추론 깊이 제어"를 동시 최적화해야 함을 보여준다.
- 핵심 기여: Reasoning RAG 설계공간을 System1/2 축으로 정리
- 키워드: #ReasoningRAG #AIAgent #ToolUse #Evaluation #IndustryAI

### 5) RAG Playground (arXiv:2412.12322)
- 링크: https://arxiv.org/abs/2412.12322
- DOI: https://doi.org/10.48550/arXiv.2412.12322
- 요약: 벡터검색·재순위화·하이브리드검색과 프롬프트 전략(ReAct 포함)을 교차 실험하는 오픈 프레임워크를 제시한다. 단일 리트리버 개선보다 검색 조합과 자기평가 프롬프트가 함께 작동할 때 성능 상승 폭이 커진다는 결과를 보이며, RAG 품질을 정답률 하나로 환원하지 않고 다중 지표로 측정한다. 에이전트형 RAG의 반복 실행 비용과 토큰 사용량을 성능 지표와 함께 다룬 점이 실무 친화적이다. 다만 WIP 논문이므로 벤치마크 안정성은 후속 업데이트 확인이 필요하다(불확실성).
- 핵심 기여: RAG 검색·프롬프트 조합의 체계 평가 도구 공개
- 키워드: #RAGEvaluation #PromptEngineering #ReAct #HybridSearch #DataScience

### 6) Conflict-Aware Soft Prompting for RAG (arXiv:2508.15253)
- 링크: https://arxiv.org/abs/2508.15253
- DOI: https://doi.org/10.48550/arXiv.2508.15253
- 요약: 외부 검색 문맥과 모델 파라메트릭 지식이 충돌할 때 발생하는 context-memory conflict를 해결하기 위해 CARE를 제안한다. 문맥 평가기가 신뢰도 신호를 생성해 소프트 프롬프트로 추론 경로를 유도하며, QA·팩트체크에서 평균 5% 개선을 보고한다. 이는 "무조건 더 많이 검색"이 아니라 "어떤 컨텍스트를 믿을지"를 학습하는 방향으로 RAG를 확장한다는 점에서 의미가 크다. 신뢰성 중심 RAG에서 안전성·환각 저감 설계의 실증적 단서를 제공한다.
- 핵심 기여: 컨텍스트-메모리 충돌 완화용 신뢰도 유도 프롬프트
- 키워드: #RAG #AgentMemory #TrustworthyAI #Prompting #FactChecking

### 7) ARAG for Personalized Recommendation (arXiv:2506.21931)
- 링크: https://arxiv.org/abs/2506.21931
- DOI: https://doi.org/10.48550/arXiv.2506.21931
- 요약: 개인화 추천에서 장기/세션 선호를 동시에 반영하기 위해 사용자이해·NLI·문맥요약·순위화의 4개 에이전트를 둔 Agentic RAG를 제시한다. 정적 휴리스틱 검색 기반 대비 NDCG@5, Hit@5 개선을 보고하며, 추천 파이프라인에 추론 가능한 근거를 부착하는 장점이 있다. 데이터셋 특성 편향이 일반화에 미치는 영향은 별도 검증이 필요하지만(불확실성), 도메인별 도구 라우팅과 메모리 계층화가 추천 시스템에서도 효과적임을 보였다.
- 핵심 기여: 추천 시스템에 에이전트형 검색·추론 협업 도입
- 키워드: #AgenticRAG #Recommendation #NLI #UserModeling #AIAgent

### 8) A-RAG: Hierarchical Retrieval Interfaces (arXiv:2602.03442)
- 링크: https://arxiv.org/abs/2602.03442
- DOI: https://doi.org/10.48550/arXiv.2602.03442
- 요약: A-RAG는 키워드 검색·의미 검색·청크 읽기를 계층 인터페이스로 노출해, 모델이 검색 단위를 동적으로 선택하도록 만든다. 기존 RAG의 고정 파이프라인을 탈피해 모델 개선이 검색 전략 개선으로 직접 이어지게 설계한 점이 핵심이다. 다수 오픈도메인 QA에서 동등/낮은 검색 토큰으로 성능 향상을 보고해 효율성-정확도 균형 가능성을 보여준다. 컨텍스트 엔지니어링 관점에서는 retrieval API 설계 자체가 모델 성능 상한을 좌우한다는 실무 메시지를 준다.
- 핵심 기여: 검색을 모델-주도 의사결정으로 전환한 계층형 RAG 인터페이스
- 키워드: #HierarchicalRetrieval #AgenticRAG #ContextEngineering #ToolUse #QASystems

### 9) Developer Challenges in AI Agent Systems (arXiv:2510.25423)
- 링크: https://arxiv.org/abs/2510.25423
- DOI: https://doi.org/10.48550/arXiv.2510.25423
- 요약: Stack Overflow와 GitHub 이슈를 결합 분석해 에이전트 개발 난제를 환경/의존성, 검색·임베딩·메모리, 오케스트레이션 제어, 모델-도구 계약, 운영 신뢰성의 5개 상위 범주로 통합한다. 흥미로운 점은 설치·프롬프트 이슈는 해결이 빠르지만, 검색·오케스트레이션 문제는 장기 미해결 비율이 높다는 결과다. 이는 벤치마크 성능보다 운영 신뢰성(tool reliability)과 디버깅 관측성의 중요성을 데이터 기반으로 뒷받침한다.
- 핵심 기여: 실무자 데이터 기반의 에이전트 엔지니어링 난제 택소노미
- 키워드: #ToolReliability #AgentOps #DeveloperExperience #AgentMemory #Orchestration

### 10) Bias in the Picture: VLM Benchmark (arXiv:2509.19659)
- 링크: https://arxiv.org/abs/2509.19659
- DOI: https://doi.org/10.48550/arXiv.2509.19659
- 요약: 뉴스 이미지 1,343쌍 기반 벤치마크로 VLM의 사회적 단서 편향을 평가하고, LLM-as-judge와 인간 검증을 결합한 평가 루브릭을 제안한다. 시각 맥락이 개방형 응답의 편향 방향을 체계적으로 이동시킬 수 있으며, faithful한 설명이 항상 낮은 편향을 뜻하지는 않는다는 결과를 제시한다. 멀티모달 에이전트가 외부 도구·검색과 결합될수록 이러한 편향 전파 위험이 커질 수 있어, 안전성 평가를 기능 성능과 분리하지 말아야 함을 시사한다.
- 핵심 기여: 멀티모달 편향 평가를 위한 재현 가능한 데이터·루브릭 제시
- 키워드: #MultimodalAgents #AISafety #VLM #BiasEvaluation #LLMasJudge
