## 1. Retrieval-Augmented LLM Agents: Learning to Learn from Experience
- **저자:** Thomas Palmeira Ferraz et al.
- **게재일:** 2026-03-18 | **분류:** cs.AI, cs.CL
- **DOI:** [10.48550/arXiv.2603.18272](https://doi.org/10.48550/arXiv.2603.18272)
- **요약:** LLM 에이전트의 일반화 성능을 향상시키기 위해 SFT(LoRA 기반)와 경험 검색(experience retrieval)을 결합하는 새로운 파이프라인을 제안. 기존 단순 fine-tuning은 새로운 태스크에 대한 외推 성능이 부족하고, 검색 기반 방법은 supervised baseline 대비 성능이 저하되는 한계를 극복함. 경험의 저장·쿼리·궤적 선택에 대한 상세 설계 분석을 통해 최적 전략을 식별하고, unseen task에 대한 일반화를 크게 개선함.
- **핵심기여:** SFT + in-context experience retrieval 통합 파이프라인; storage/querying/trajectory selection 설계 가이드라인 제시
- **링크:** <https://arxiv.org/abs/2603.18272>
- **해시태그:** #LLMAgent #ExperienceRetrieval #SFT #Generalization #LoRA

## 3. From Storage to Steering: Memory Control Flow Attacks on LLM Agents
- **저자:** Zhenlin Xu et al.
- **게재일:** 2026-03-16 | **분류:** cs.CR
- **DOI:** [10.48550/arXiv.2603.15125](https://doi.org/10.48550/arXiv.2603.15125)
- **요약:** LLM 에이전트의 메모리 검색이 컨트롤 플로우를 지배하여 의도하지 않은 도구 사용을 강제하는 Memory Control Flow Attacks (MCFA)를 최초로 식별. MEMFLOW라는 자동화 평가 프레임워크를 설계하여, LangChain과 LlamaIndex 기반의 실제 도구 환경에서 GPT-5 mini, Claude Sonnet 4.5, Gemini 2.5 Flash 등 최신 모델을 공격. 90% 이상의 시도에서 취약성이 확인됨.
- **핵심기여:** MCFA 새로운 공격 벡터 발견; MEMFLOW 자동 평가 프레임워크; 주요 LLM 프레임워크에 대한 광범위 취약성 입증
- **링크:** <https://arxiv.org/abs/2603.15125>
- **해시태그:** #AgentSecurity #MemoryAttack #MCFA #LLMSafety #PromptInjection

## 4. Efficient LLM Serving for Agentic Workflows: A Data Systems Perspective (Helium)
- **저자:** Yao Lu et al.
- **게재일:** 2026-03-17 | **분류:** cs.MA, cs.AI, cs.DB
- **DOI:** [10.48550/arXiv.2603.16104](https://doi.org/10.48550/arXiv.2603.16104)
- **요약:** 에이전트 워크플로우의 중복 프롬프트와 중간 결과에 기인한 효율성 문제를 데이터 시스템 관점에서 해결. Helium 프레임워크는 에이전트 워크로드를 쿼리 플랜으로 모델링하고, LLM 호출을 일급 연산자로 취급하여 proactive caching과 cache-aware scheduling을 적용. 기존 최고 성능 대비 최대 1.56배 속도 향상 달성.
- **핵심기여:** 워크플로우 차원의 엔드투엔드 최적화; 크로스-콜 종속성 활용; 대규모 에이전트 서빙 효율화
- **링크:** <https://arxiv.org/abs/2603.16104>
- **해시태그:** #LLMServing #AgenticWorkflows #Caching #DataSystems #Efficiency
