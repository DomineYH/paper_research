### 1) Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned
- 링크: https://arxiv.org/abs/2603.05344v1
- PDF: https://arxiv.org/pdf/2603.05344v1.pdf
- DOI: 확인 불가(미기재)
- 요약(약 500자): 터미널 네이티브 코딩 에이전트가 IDE 플러그인 중심 접근을 대체하는 흐름에서, 본 논문은 장기 과업 수행을 위한 안전한 구조를 체계화한다. 핵심은 계획/실행 분리형 듀얼 에이전트, 작업별 모델 라우팅, 지연형 도구 탐색, 적응형 컨텍스트 압축, 이벤트 기반 리마인더를 결합한 복합 아키텍처다. 특히 긴 세션에서 발생하는 컨텍스트 팽창과 지시 희석(instruction fade-out)을 실무 문제로 정의하고, 메모리 축적 및 압축 전략으로 완화한다. 에이전트 엔지니어링을 ‘모델 성능’이 아니라 ‘시스템 설계’ 문제로 재정의했다는 점에서 실용적 기여가 크다.
- 핵심 기여:
  - 터미널 우선 에이전트 설계 원칙 정리
  - 컨텍스트 엔지니어링(압축/리마인드/메모리)의 시스템화
  - 안전성과 확장성을 동시에 고려한 오픈소스 청사진 제시
- 키워드: #PromptEngineering #ContextEngineering #AIAgent #AgentMemory #ToolReliability

### 2) Leveraging LLM Parametric Knowledge for Fact Checking without Retrieval
- 링크: https://arxiv.org/abs/2603.05471v1
- PDF: https://arxiv.org/pdf/2603.05471v1.pdf
- DOI: 확인 불가(미기재)
- 요약: 검색 기반 검증(RAG) 의존에서 벗어나, LLM 내부 파라메트릭 지식을 이용한 비검색 사실검증을 제안한다. 저자들은 장꼬리 지식, 출처 다양성, 다국어성, 장문 생성 조건에서 일반화를 점검하는 평가 프레임워크를 구성하고 9개 데이터셋에서 18개 방법을 비교했다. 결과적으로 단순 logit 기반보다 내부 표현 상호작용을 활용한 접근이 더 강건하며, 제안법(INTRA)이 높은 일반화 성능을 보였다. 이는 RAG의 검색 오류를 회피하는 보완축을 제공하며, 보상모델/생성중 검증 컴포넌트로의 통합 가능성을 제시한다.
- 핵심 기여:
  - Retrieval-free fact checking 과업 정식화
  - 다면 일반화 평가 체계 구축
  - 내부표현 기반 방법(INTRA)로 SOTA 보고
- 키워드: #FactChecking #RAGEvaluation #LLM #TrustworthyAI #DataScience

### 3) Towards Provably Unbiased LLM Judges via Bias-Bounded Evaluation
- 링크: https://arxiv.org/abs/2603.05485v1
- PDF: https://arxiv.org/pdf/2603.05485v1.pdf
- DOI: 확인 불가(미기재)
- 요약: LLM-as-a-Judge가 자율 피드백 루프의 핵심 인프라가 되는 상황에서, 미지/적대적 편향 조건에서도 통제 가능한 평가 프레임워크(A-BB)를 제안한다. Arena-Hard-Auto 환경에서 4개 judge를 대상으로 포맷/스키마 편향을 검증한 결과, 편향 제한 보장을 유지하면서도 원래 순위와의 상관을 유의미하게 유지했다. 메시지는 명확하다. ‘평균 성능’만으로 judge를 선택하면 위험하며, 운영단에서는 편향 상한을 명시한 검증 규약이 필요하다. AI safety-evals와 배포 거버넌스 간 간극을 메우는 연구다.
- 핵심 기여:
  - 평균 편향 제한(A-BB) 개념과 알고리즘 제시
  - 편향 통제와 순위보존 간 실증적 트레이드오프 제시
  - 평가기 신뢰성의 보증 가능성 논의 확장
- 키워드: #AISafety #Evals #LLMJudge #Bias #ToolReliability

### 4) Judge Reliability Harness: Stress Testing the Reliability of LLM Judges
- 링크: https://arxiv.org/abs/2603.05399v1
- PDF: https://arxiv.org/pdf/2603.05399v1.pdf
- DOI: 확인 불가(미기재)
- 요약: 본 연구는 LLM 평가기의 견고성을 스트레스 테스트하는 오픈소스 하네스를 제안한다. 단순 정확도 외에도 자유서술·에이전트 과업 형식에서 이진 판단과 서열 채점의 일관성을 함께 검증하도록 설계되었다. 안전·설득·오남용·에이전트 행동 벤치마크에서 최신 judge들을 비교한 결과, 모델과 섭동 유형에 따라 신뢰성이 크게 달라졌으며, 문장 재표현·포맷 변화·정답 라벨 전환 같은 경미한 변화에도 판정이 흔들리는 문제가 확인됐다. 운영 환경의 judge calibration과 회귀 테스트 자동화 필요성을 강하게 뒷받침한다.
- 핵심 기여:
  - LLM judge 전용 reliability harness 공개
  - 섭동 기반 일관성 검증 프로토콜 제안
  - 실제 벤치마크에서 비균질 취약성 실증
- 키워드: #LLMJudge #Reliability #AISafety #EvaluationHarness #AIAgent

### 5) Distributed Partial Information Puzzles: Examining Common Ground Construction Under Epistemic Asymmetry
- 링크: https://arxiv.org/abs/2603.05450v1
- PDF: https://arxiv.org/pdf/2603.05450v1.pdf
- DOI: 확인 불가(미기재)
- 요약: 다중 참여자가 서로 다른 정보를 가진 상태에서 공통기반(common ground)을 형성하는 문제를 DPIP 과업으로 정식화했다. 음성·제스처·행동이 시간 정렬된 멀티모달 데이터셋을 구축하고, LLM 기반 추론과 DEL(동적 인식논리) 기반 공리적 파이프라인을 비교했다. 결과는 현대 LLM이 과업 진행 추적뿐 아니라 신념 상태 업데이트에서도 취약함을 보여준다. 이는 멀티모달 에이전트가 협업 맥락을 다룰 때 필요한 내부 상태 표현, 명시적 신념 추적, 상호확인 프로토콜 설계의 중요성을 환기한다.
- 핵심 기여:
  - epistemic asymmetry 환경의 협업 벤치마크 제시
  - 멀티모달 신념 추적 데이터셋 제공
  - LLM vs 논리기반 접근의 성능 간극 실증
- 키워드: #MultimodalAgents #CommonGround #AIAgent #ContextEngineering #Evaluation

### 6) Ensembling Language Models with Sequential Monte Carlo
- 링크: https://arxiv.org/abs/2603.05432v1
- PDF: https://arxiv.org/pdf/2603.05432v1.pdf
- DOI: 확인 불가(미기재)
- 요약: 다양한 모델·프롬프트 조합의 민감도를 완화하기 위해 문자열 분포 수준의 앙상블을 정식화하고, 바이트 단위 SMC 샘플러를 제안한다. 기존 next-token 평균이 유도하는 국소 정규화 편향을 지적하며, 서로 다른 vocabulary를 가진 모델들도 공통 문자 공간에서 일관되게 결합할 수 있게 했다. 구조화 텍스트 생성에서 단순 평균 대비 우수한 집계 함수가 존재함을 보이고, posterior 근사 품질이 실제 생성 성능으로 이어짐을 실험적으로 확인했다. 프롬프트 민감성·재현성 문제에 대한 데이터과학적 대안으로 볼 수 있다.
- 핵심 기여:
  - 일반화된 f-ensemble 분포 프레임워크 제시
  - byte-level SMC로 이질 vocab 모델 결합
  - 근사 품질과 다운스트림 성능 연계 실증
- 키워드: #Prompting #DataScience #Ensembling #LLM #Evaluation

- PDF: https://arxiv.org/pdf/2603.05451v1.pdf

- 요약: Blackwell 세대 GPU의 비대칭 하드웨어 스케일링(텐서코어 급증 vs 기타 유닛 상대 정체)에 맞춘 attention 최적화 기법을 제안한다. 비동기 MMA 파이프라인, 소프트웨어 지수 연산, tensor memory/2-CTA 모드 활용으로 병목을 재배치해 처리량을 크게 높였다. 인프라 관점에서는 긴 컨텍스트 처리 비용을 낮춰 context engineering 실험 가능성을 확대한다. 또한 CuTe-DSL 기반 구현으로 컴파일 시간을 크게 줄여 연구-운영 전환 속도를 높였다.
- 핵심 기여:
  - Blackwell 특화 attention 병목 재설계
  - 성능 및 컴파일 생산성 동시 개선
  - 장문맥/에이전트 워크로드 실용성 강화
- 키워드: #LongContext #SystemsForAI #ContextEngineering #LLMInfra #Efficiency

### 8) RelaxFlow: Text-Driven Amodal 3D Generation
- 링크: https://arxiv.org/abs/2603.05425v1
- PDF: https://arxiv.org/pdf/2603.05425v1.pdf
- DOI: 확인 불가(미기재)
- 요약: 부분 관측 이미지에서 텍스트 프롬프트로 가려진 3D 영역을 보완 생성하는 문제를 제시한다. 관측 보존은 강한 제어, 프롬프트 반영은 완화 제어가 필요하다는 점을 분리해 Multi-Prior Consensus와 Relaxation 메커니즘을 결합했다. 이론적으로 완화 연산이 생성 벡터장의 저역통과 필터와 동치임을 보이며, 구조 정보 보존과 인스턴스 세부 억제를 설명한다. 멀티모달 에이전트의 지각-추론-생성 루프에서 프롬프트 제어성과 관측 충실도 간 균형 설계에 참고가 된다.
- 핵심 기여:
  - text-driven amodal 3D 생성 과업 정식화
  - 제어 강도 분리형 training-free 프레임워크
  - 진단 벤치마크(ExtremeOcc-3D, AmbiSem-3D) 제안
- 키워드: #Multimodal #PromptControl #3DGeneration #AIAgentPerception #Evaluation

### 10) SurvHTE-Bench: A Benchmark for Heterogeneous Treatment Effect Estimation in Survival Analysis
- 링크: https://arxiv.org/abs/2603.05483v1
- PDF: https://arxiv.org/pdf/2603.05483v1.pdf
- DOI: 확인 불가(미기재)
- 요약: 검열(censoring)된 생존데이터에서 이질적 처치효과(HTE) 추정을 평가하는 통합 벤치마크를 제안한다. 합성·반합성·실데이터를 포괄하고 인과 가정 위반 상황까지 포함해, 기존 파편화된 비교 관행을 정리했다. 본 주제군(프롬프트/에이전트)과 직접적 거리는 있으나, ‘평가 프로토콜의 엄밀성’이라는 공통 축에서 AI eval 설계에 시사점을 준다. 즉, 에이전트 평가에서도 가정 명시·오류모드 분해·재현 가능한 테스트 매트릭스가 필수임을 확인시킨다.
- 핵심 기여:
  - 생존 HTE 방법의 공정·재현 비교 프레임 제공
  - 가정 위반을 포함한 현실적 평가 시나리오 설계
  - 벤치마크 문화의 표준화 기여
- 키워드: #Evaluation #Benchmark #DataScience #CausalML #Reproducibility
